# Deep Explanation: Conhecendo a Vercel

## Por que a Vercel?

A Vercel e uma das plataformas mais utilizadas para deploy de aplicacoes React. O instrutor destaca dois motivos principais:

1. **Popularidade** — ser amplamente adotada pela comunidade significa mais documentacao, tutoriais e suporte disponivel
2. **Plano gratuito (Hobby)** — permite testar e publicar aplicacoes sem custo, removendo barreiras para aprendizado

## GitHub como estrategia de autenticacao

O instrutor recomenda "fortemente" conectar via GitHub em vez de usar email. A razao e pratica: dentro da Vercel, o fluxo de deploy depende do GitHub para importar repositorios. Usar GitHub desde o inicio evita configuracoes extras depois.

### Fluxo de autenticacao

1. Clicar em Sign Up
2. Selecionar Hobby (projetos pessoais)
3. Informar nome
4. Escolher "Continue with GitHub"
5. Se ja esta logado no GitHub no navegador, a autenticacao e automatica
6. Dashboard carrega com projetos (ou vazio na primeira vez)

## Planos da Vercel

| Plano | Uso | Custo |
|-------|-----|-------|
| Hobby | Projetos pessoais, aprendizado, testes | Gratuito |
| Pro | Projetos profissionais, equipes pequenas | Pago |
| Enterprise | Escala, compliance, SLA | Pago (custom) |

O instrutor enfatiza que o plano Hobby e suficiente para o curso inteiro. A transicao para planos pagos so faz sentido quando o projeto precisa escalar ou ser monetizado.

## Contexto no fluxo de deploy

Esta aula e preparatoria — foca exclusivamente na criacao da conta. O deploy propriamente dito acontece nas aulas seguintes. O instrutor pede explicitamente para pausar e criar a conta antes de continuar, indicando que as proximas aulas dependem de ter a Vercel configurada.
