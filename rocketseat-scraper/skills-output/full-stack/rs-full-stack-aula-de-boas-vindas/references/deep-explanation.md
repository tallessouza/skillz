# Deep Explanation: Sistema de Reembolso — Aula de Boas Vindas

## Abordagem Pedagógica

O instrutor posiciona este projeto como um "quebra-cabeça": cada conceito JavaScript aprendido anteriormente é uma peça individual, e agora o objetivo é juntar todas as peças em uma aplicação funcional.

### Por que HTML e CSS são fornecidos prontos

O instrutor explicitamente remove HTML e CSS do escopo para manter o foco no JavaScript. Isso reflete uma prática pedagógica importante: isolar a variável de aprendizado. Quando o aluno precisa fazer tudo ao mesmo tempo (markup, estilo e lógica), o aprendizado de cada parte é diluído.

### Filosofia de integração

A proposta não é aprender conceitos novos, mas sim aprender a **integrar** conceitos já conhecidos. Essa é uma habilidade distinta — saber usar `addEventListener`, manipular DOM, trabalhar com arrays e objetos individualmente é diferente de saber combinar tudo em um fluxo coerente.

### O que esperar do módulo

- Sistema de reembolso completo com interface funcional
- JavaScript como única linguagem de implementação
- Aplicação prática de conceitos intermediários de JS
- Foco em dar "vida" a uma interface estática

## Contexto do módulo

Este é o módulo "JavaScript Intermediário" do curso Full Stack da Rocketseat. A etapa anterior cobriu fundamentos de JS e esta etapa aplica esses fundamentos em projeto real.
