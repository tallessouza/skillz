# Deep Explanation: Conhecendo o Projeto

## Contexto do instrutor

O instrutor apresenta um projeto que sera desenvolvido ao longo do modulo: uma landing page completa + blog, usando Next.js com Pages Router. O design foi criado pela designer Milena e esta disponivel no Figma.

## Por que landing page + blog?

A combinacao e estrategica para o aprendizado:
- **Landing page** exercita componentizacao, responsividade, e uso intensivo dos recursos do Next.js (especialmente relacionados a renderizacao e roteamento)
- **Blog** e um caso de uso real e pratico — o instrutor menciona que serve para quem quer comecar a escrever seus proprios posts, tirando a ideia do papel

## Filosofia de projeto limpo

O instrutor enfatiza a importancia de limpar o projeto antes de comecar:
- Remove `console.log` de aulas anteriores
- Remove pastas que nao serao utilizadas
- Remove imports desnecessarios (como `Image` nao utilizado)

Isso reflete uma pratica profissional: nunca construa sobre codigo sujo. O estado inicial do projeto deve ser um "canvas limpo".

## Observacoes sobre o design

O instrutor destaca:
- Design "clean" — minimalista e bem estruturado
- Secoes bem definidas: header, features com icones, CTA, testimonials
- Parte responsiva (mobile) ja contemplada no design
- O design no Figma e a referencia principal para toda implementacao

## Recursos do Next.js que serao explorados

O instrutor menciona que "todo esse conhecimento que a gente vem adquirindo ate aqui sobre o Next vai ser muito utilizado", indicando que o projeto consolida:
- File-based routing (Pages Router)
- Componentizacao
- Recursos de renderizacao do Next.js
- Otimizacao de assets
