# Deep Explanation: Análise Combinatória — Introdução

## O que é análise combinatória

Conforme o instrutor Rodolfo explica, análise combinatória é uma área da matemática onde se estuda como **selecionar, agrupar e organizar** certos elementos, conjuntos de dados e informações.

O ponto-chave da introdução é que a combinatória não é sobre listar possibilidades manualmente — é sobre ter **ferramentas** que permitem calcular o número total de possibilidades de forma sistemática.

## Por que isso importa para analistas de dados

O instrutor destaca dois cenários práticos:

### 1. Seleção de grupos para teste A/B
Quando um analista precisa selecionar grupos de pessoas para um teste A/B, está implicitamente usando análise combinatória. Quantos grupos distintos podem ser formados? Qual a probabilidade de um grupo ser representativo? Essas perguntas dependem de combinatória.

### 2. Apresentação de produtos em campanhas
O exemplo concreto: dados 3 produtos (A, B, C), de quantas formas diferentes podem ser apresentados em uma campanha? O instrutor lista: ABC, BAC, CBA... mas o ponto é que a combinatória fornece a fórmula (3! = 6) em vez de exigir enumeração manual.

## Estrutura do módulo

O módulo é dividido em duas partes:
1. **Combinatória** — técnicas de contagem, seleção e organização
2. **Probabilidade** — construída sobre a base combinatória

Essa separação é importante porque probabilidade depende de combinatória. Entender quantos resultados possíveis existem (combinatória) é pré-requisito para calcular a chance de um resultado específico (probabilidade).
