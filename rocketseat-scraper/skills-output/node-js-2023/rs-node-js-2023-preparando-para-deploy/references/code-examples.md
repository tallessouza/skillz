# Code Examples: Preparando para Deploy

## Tentativa com tsc (o que NAO fazer)

### Configuracao no tsconfig.json

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./build"
  }
}
```

### Execucao

```bash
npx tsc
```

**Resultado:** Erro "TransactionsSpec is not under rootDir" + arquivos .js gerados em locais indesejados (test/, raiz).

## Build com tsup (o correto)

### Instalacao

```bash
npm install tsup -D
```

### Script no package.json

```json
{
  "scripts": {
    "build": "tsup src --out-dir build"
  }
}
```

### Execucao

```bash
npm run build
```

**Resultado:** Build em ~14ms, pasta `build/` com apenas os arquivos de `src/` convertidos para CommonJS.

### Verificacao

```bash
node build/server.js
```

O servidor deve iniciar normalmente, sem dependencia de TypeScript.

## Flags do tsup

```bash
# Descobrir todas as opcoes
npx tsup -h

# Especificar diretorio de saida
tsup src --out-dir build   # forma longa
tsup src -d build          # forma curta
```

## Fluxo completo de upload para GitHub

```bash
# Iniciar git (se nao iniciado)
git init

# Commit inicial
git add .
git commit -m "initial commit"

# Criar repo e push via GitHub CLI
gh repo create
# Opcoes interativas:
# - Push an existing local repository to GitHub
# - Path: .
# - Nome do repositorio: nome-do-projeto
# - Visibilidade: private (ou public)
# - Add remote: yes
# - Remote name: origin
# - Push commits: yes

# Verificar no browser
gh repo view -w
```

## Limpeza de arquivos .js acidentais do tsc

Se `tsc` foi executado por engano e gerou arquivos .js:

```bash
# Verificar arquivos indesejados
git status

# Remover arquivos .js gerados fora de build/
# (ex: test/*.js, vitest.config.js gerado)
rm test/**/*.js
rm vitest.config.js  # se gerado pelo tsc

# Commitar a limpeza
git add .
git commit -m "fix: remove js files generated by tsc"
git push origin main
```
