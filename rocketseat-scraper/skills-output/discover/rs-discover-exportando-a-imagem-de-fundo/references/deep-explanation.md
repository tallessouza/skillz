# Deep Explanation: Exportando Assets do Figma

## Por que uma pasta assets dedicada?

O instrutor (Mayk) enfatiza a organizacao desde o inicio do projeto. A pasta `assets/` serve como ponto central para todas as imagens do projeto, separando conteudo visual do codigo.

Isso evita:
- Imagens soltas na raiz do projeto
- Caminhos quebrados quando o projeto cresce
- Confusao entre arquivos de codigo e arquivos de midia

## Duas abordagens — e por que ambas sao validas

O Mayk apresenta duas opcoes sem forcar uma escolha:

1. **Baixar a pasta Assets completa** dos materiais complementares da Rocketseat. Isso ja traz todas as imagens que serao usadas ao longo do modulo. Vantagem: nao precisa parar para exportar em cada aula futura.

2. **Exportar uma a uma do Figma** usando o atalho `Ctrl+Shift+E` (Windows) ou `Shift+Cmd+E` (Mac). Isso abre o painel lateral de exportacao do Figma, onde cada asset tem um botao de export individual.

A escolha e do aluno. O importante e que a pasta `assets/` exista e as imagens estejam la.

## O painel de exportacao do Figma

No Style Guide do projeto Figma, os designers ja deixaram os assets marcados para exportacao. O atalho abre um painel lateral que lista todos os elementos exportaveis com seus nomes (ex: `BG Mobile`).

Ao clicar em "Export", o Figma salva o arquivo na pasta de downloads padrao do sistema operacional. O aluno entao precisa mover manualmente para a pasta `assets/` do projeto.

## Organizacao progressiva

O Mayk menciona que nas proximas aulas vai continuar exportando imagens e adicionando a pasta. Isso mostra um padrao incremental — a pasta `assets/` vai crescendo conforme o projeto avanca. Se o aluno baixou tudo de uma vez, ja esta pronto. Se nao, vai adicionando aula a aula.

## Extraindo arquivos zipados

Se o aluno baixou a pasta Assets dos materiais complementares, ela vem compactada. O Mayk lembra de extrair o conteudo antes de usar — um erro comum de iniciantes e tentar referenciar arquivos dentro de um .zip.
