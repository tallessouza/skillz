WEBVTT Na aula de hoje nós vamos falar sobre Arrays. Arrays são bem parecidos como em qualquer outra linguagem de programação é uma estrutura de dados que existe em Go que basicamente representa um espaço contínuo de memória. O Array armazena em memória vários elementos do mesmo tipo só que em Go Array tem algumas peculiaridades. Então primeiro vamos ver a sintaxe de um Array. Para declarar um Array a gente pode declarar tanto usando a declaração completa usando o var, a palavra-chave, o tipo, o valor ou a gente pode usar o short statement também. Então eu vou usar o short statement aqui só para ser mais rápido e vou dizer que o nome da minha variável é arr para Array. E aqui a gente vai entender o que é o tipo Array. Um Array em Go tem que ter o tamanho dele sabido em tempo de compilação ou seja o tamanho dele tem que ser constante. Então por exemplo se eu for criar um Array de inteiros eu tenho que colocar aqui quantos inteiros vão ter dentro desse Array. Então vamos supor que eu queira um Array de 3 inteiros. Vou colocar aqui colchetes, 3, fecha colchetes, o tipo e aqui os valores. Eu posso deixar sem nenhum valor lá dentro e aí eu vou ter um Array de 3 inteiros em que os 3 valores estão zerados. Se eu vier aqui e imprimir esse valor para mostrar isso para vocês eu posso rodar esse nosso código e vai estar lá 0 0 0 mas eu também posso inicializar os valores. Então por exemplo quero 1, 2, 3 isso aqui também é válido, não tem problema nenhum. Se eu rodar o código está lá 1, 2, 3 e eu posso deixar alguns valores sem serem definidos. Então por exemplo eu posso definir só o primeiro valor e aí vai estar 1 0 0 porque os outros valores vão ser zerados. O que eu não posso fazer é colocar um elemento a mais. Então eu não posso fazer 1, 2, 3, 4. Isso aqui vai ser um erro de compilação. Já está lá o nosso inter dizendo que o index é out of bounds. Esse código é o mesmo erro, é um erro de compilação não é um erro de runtime. E outra coisa que dá pra gente fazer também é especificar em qual índice a gente quer um elemento. Então por exemplo, se eu vier aqui e vou apagar isso tudo, vamos dizer que eu quero um Array de 10 elementos. E no índice 5 eu quero o valor 400. Se eu vier aqui e salvar esse código, não tem nenhum erro. E se eu rodar esse código, está lá o nosso sexto elemento, ou seja, o elemento do índice 5 está com o valor 400. Isso também é válido. Se eu quiser especificar mais do que um elemento, eu posso colocar uma vírgula. Então por exemplo, pode dizer que no elemento do índice 7 vai ser o valor 300. Isso aqui também é válido, não tem problema nenhum. E claro, esses Arrays podem ser de qualquer tipo em Go. Então se eu vier aqui e apagar isso aqui, vou colocar um Array de Strings, por exemplo, que vão ser 10 Strings. E no índice 4 eu vou dizer que é a String Hello World. Isso aqui também é válido, não tem problema nenhum. No caso ali, no nosso terminal, a gente está vendo esses espaços em brancos, porque são Strings vazias, sem nada dentro delas. E uma coisa importante saber, como eu disse, o tamanho do Array tem que ser constante, ele tem que ser sabido em tempo de compilação. Então eu não posso, por exemplo, vir aqui e declarar uma variável X, que o tamanho é 10, e passar essa variável aqui para dentro do nosso Array. Isso aqui é um erro de compilação. Não é permitido fazer isso, porque X não é constante. Mas se eu transformar X em uma constante, aí tudo bem, não tem problema nenhum eu fazer isso. E além disso, além do tamanho dos Arrays terem que ser sabidos em tempo de compilação e terem que ser constantes, ele nunca pode mudar de tamanho. Então eu não posso aumentar o tamanho desse Array, nem diminuir o tamanho desse Array. Ele é sempre desse tamanho. Esse é o conceito de Arrays em Go. É claro que a gente tem Arrays dinâmicos, que em Go são chamados de Slice, que são Arrays que têm tamanho dinâmico. Eles podem crescer de tamanho. Mas isso é um papo para uma outra aula.
