WEBVTT Fala Dev, beleza? Bem-vindo a mais um módulo da trilha de Go. Depois de me apresentar, e agora que a gente se conhece um pouquinho mais, eu vou falar sobre o próximo projeto que a gente vai construir nesse módulo. Diferente dos módulos anteriores, que eram mais teóricos, chegou a hora da gente pôr a mão na massa e construir um projeto do zero. O projeto em questão é o GoBid. Ele vai ser uma auction house bem parecida com o eBay. A gente vai usar esse projeto para aprender um pouquinho mais sobre WebSocket, Chat Sessions, Password Hashing, Segurança com o CSRF Tokens. A gente vai usar também algumas ferramentas que aprendemos nos últimos módulos, como o SKLC, Tern, Docker, e uma ferramenta nova muito legal que vai ajudar a gente a construir qualquer aplicação em Go, que ela se chama R. É um Live Reloading em Go. Fora isso, a gente também vai ter uma introdução básica com channels. Mas nada para se assustar, ok? A gente vai ter um conjunto de aulas específicas sobre channels nos próximos módulos, mas aqui a gente vai ter algo para se familiarizar. Esse projeto também vai apresentar o básico da estrutura de projetos que você pode utilizar. Mas da mesma forma que o uso de channels vai servir apenas como uma introdução e familiarização com o tópico, esse projeto vai apenas mostrar uma das várias formas que você pode utilizar num projeto em Go. Isso vai ter o intuito de você já ter alguma experiência para quando chegar no momento de falar sobre isso especificamente, em algum módulo, você já vai ter um pensamento formado com a capacidade de escolher o melhor por momento ou o projeto que você está contribuindo ou construindo. Dentro desse projeto, a gente vai imaginar que a gente tenha um usuário que interaja com algum front-end, com algum cliente. Isso pode ser um front-end escrito em React, uma interface de um app ou qualquer coisa que você queira imaginar. Mas a gente não vai fazer o front-end, a gente vai estar apenas aplicando o back-end aqui. O back-end vai ser a nossa API. A nossa API vai ter um router que vai lidar com duas rotas principais. A V1, que vai ter API barra V1. E a gente vai ter o WebSocket. Vamos dar uma olhada na V1, que a gente vai ter o WebSocket. Vamos dar uma olhada na V1 que a gente vai ter ali de importante. Primeiro, a gente vai ter a rota de users que vai interagir com o user service que interage com o PostgreSQL Database. Por que a gente vai ter um serviço aqui? Para aplicar qualquer coisa que seja de regra de negócio. Da mesma forma com o produto. Por exemplo, nos usuários a gente vai ter que ser capaz de criar, deletar, logar, deslogar usuários. E isso tudo precisa ser guardado dentro do post GrassQL, lidar com sessões, etc. Dentro da rota de produtos a gente vai ter a capacidade de criar produtos, caso seja um usuário autenticado, e aí a gente vai falar de sessões, autenticação e autorização. Toda a interação com o banco de dados vai ser feita através do user service, product service, ou seja lá qual for o service com o banco de dados. Isso apenas para a gente ter uma camada que lida com a regra de negócio, para a gente não ter todo esse código dentro dos nossos handlers em questão. Dentro do WebSocket, a gente vai ter uma parte bem específica. No WebSocket, a gente vai ter apenas uma rota. A SubscribeProductID. Dentro da SubscribeProductID, ele vai ser capaz de escutar sobre, isso é, ser capaz de tanto colocar bits, lances dentro de um produto, como ficar sabendo de bits ou apostas de outros usuários. E ele também vai ser necessário que saiba quando uma auction acabou. E isso tudo é definido na barra de produtos. Então quando uma pessoa cria um produto, o produto é criado em uma room, uma WebSocket room, uma sala, vai ser criada ao mesmo tempo que o produto. E vai existir até a deadline do mesmo. Então, a pessoa vai poder colocar lances lá, o usuário vai poder colocar lances lá, e o usuário que colocou o maior lance até o final da deadline vai ser a pessoa que ganha o produto. Então, é assim que funciona uma auction house básica. E a gente vai implementar isso em Word. Legal, né? Um projeto bem básico, mas vai te mostrar muita coisa interessante.
