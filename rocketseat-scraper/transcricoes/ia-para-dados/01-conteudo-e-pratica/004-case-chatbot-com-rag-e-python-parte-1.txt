WEBVTT Olá, boas-vindas a mais uma aula. E o foco agora é utilizar IA e uma técnica super especial chamada RAG para criar um chatbot inteligente para a área de RH. Mas antes de ir para a prática, preciso passar um pouquinho de contexto para vocês. Pasmem, 95% de todos os dados que temos no planeta terra são dados privados. E as nossas LLMs, o chat IPT, Gemini, Cloud, eles são treinados com dados públicos em sua maioria, dados disponíveis na internet. São muitos dados, é muita informação, mas se eu perguntar pra LLM como funciona férias na minha empresa, por exemplo, ela não vai conseguir responder porque ela não teve acesso a esses dados no momento do seu treinamento. Isso por si só já é um problema, já é algo ruim. Mas pode ainda piorar. Por quê? Porque a IA, a LLM, ela é treinada pra nos responder. Ela quer sim encontrar algo que se adeque, que esteja o mais próximo possível daquilo que a gente perguntou, pra que traga pra gente essa resposta e satisfaça a nossa dúvida. Pra isso, pode ser que aconteça o que a gente chama de alucinação. Que é, a AI responde de uma forma tão convincente que a gente acredita que é real. Mas não, ela está alucinando. Ela está utilizando da autoridade de super inteligência artificial. para nos passar uma resposta errada. E pior do que não ter uma resposta para a nossa pergunta é ter uma resposta errada para aquilo que a gente pergunta, sobretudo se o nosso produto é um chatbot inteligente que está em contato com nossos clientes, por exemplo. Então esse é definitivamente um problema que nós não queremos, principalmente quando estamos trabalhando com chatbot. Como nós podemos resolver isso? Eu já te respondo. Com RAG ou REG ou o nome em inglês que é Retrieval Aumented Generation. Recuperação Aumentada de Geração. Recuperação de Geração Aumentada. Basicamente, o REG vem para solucionar esse problema. Isso é, antes de responder uma pergunta, a IA vai consultar uma base de informação, uma base de conhecimento. Pode ser uma base com perguntas e respostas sobre o RH, perguntas frequentes, uma FAQ. Pode ser perguntas e respostas sobre o meu produto. Vai ser algo nem tão estruturado assim, apenas a descrição dos nossos serviços. O ponto é, antes de responder, a IAC consulta essa base de conhecimento para aí sim fornecer uma resposta para aquela pergunta que eu fiz. Por debaixo dos panos, o que funciona é A LLM recebe uma pergunta e ela traz também todo o histórico de mensagens, aqui a gente está trabalhando no contexto de um chatbot, no contexto de IA conversacional. Então ela recebe tanto a pergunta quanto o histórico de mensagens que foi trocado com a IA, com a LLM. Caso seja uma pergunta padrão, algo que apenas com o treinamento de dados públicos que a CIA recebeu ela consiga nos responder, ela apenas faz isso, ela responde, traz uma resposta para aquilo que a gente perguntou. Do contrário, se eu perguntei uma informação privada e ela tem acesso a uma base de conhecimento que possua aquelas informações, o que a IA faz é buscar nessa base de conhecimento, que geralmente é um banco de dados vetorial, faz uma busca de similaridade, o que é provável que seja mais similar em relação a significado para a minha pergunta, retorna alguns documentos, algumas informações que a LLM pode utilizar para se basear e fornecer uma resposta legal, e aí sim temos uma boa resposta. É assim que funciona. Claro que existe uma matemática mais pesada por trás de Regg, por trás de Hag, A gente não vai entrar tanto nos detalhes aqui, o foco é trabalhar a parte de IA generativa. E a gente vai utilizar Python para recriar isso que vocês estão vendo aqui na tela.
