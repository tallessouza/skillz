WEBVTT Fala, Dev! Beleza? Na aula de hoje nós iremos entender o que é uma API e qual a sua importância para nós como pessoas desenvolvedoras. Se você está começando no desenvolvimento de aplicações web ou mobile, muito provavelmente deve ter escutado sobre APIs. E bem, API basicamente significa Application Programming Interface. É uma interface que permite que sistemas diferentes se comuniquem. Ela funciona como um mensageiro, recebe um pedido do seu app ou até mesmo do seu site, leva até o servidor e traz a resposta de volta. É essencial para integrar funcionalidades como login, pagamentos, registros e leituras de dados externos em apps web ou mobile. No geral, hoje em dia, quando nós desenvolvemos algum tipo de aplicação web ou mobile, há necessidade de consumirmos algum tipo de dado externo, sejam eles dados de usuários ou dentro do contexto de aplicações bancárias, como por exemplo o saldo do usuário ou o extrato. E geralmente para nossa interface, ter esse tipo de dado é consumida diretamente de uma API, no qual realiza o meio de campo entre o cliente e o servidor. E qual a importância de nós termos esse tipo de conhecimento no nosso dia a dia? Entender os conceitos básicos de API é essencial para qualquer desenvolvedor, esse conhecimento permite uma comunicação eficiente entre sistemas, facilita integrações com serviços externos, além de ser altamente valorizado no mercado. Dominar APIs abre portas para criar soluções mais seguras, escaláveis e aliadas com demandas do desenvolvimento de software moderno. Como eu disse anteriormente, hoje em dia muitas vezes as nossas aplicações vão além do client-side. Realmente nós precisamos registrar algum tipo de dado em um banco de dados, consumir alguma foto, algum vídeo, alguma música e para isso as APIs fazem todo esse meio de campo com o nosso client-side. E para ficar mais claro o entendimento sobre API, eu irei fazer uma analogia com o restaurante. Essa analogia é bem conhecida hoje em dia. Basicamente, quando nós chegamos em um restaurante, nós, como cliente, temos um menu onde nós podemos escolher algum pedido para nós fazermos ao nosso garçom, no qual o próprio faz o meio de campo entre nós e a cozinha, onde a cozinha, de fato, é feito o nosso pedido, toda a comida é preparada. Assim que o pedido é finalizado a cozinha encaminha para o garçom e o garçom por fim traz o nosso pedido novamente no contexto que estamos abordando basicamente o cliente seria o nosso aplicativo ou o nosso site onde nós faríamos um request para a nossa API no caso seria o nosso garçom enviaríamos algum tipo de dado para a nossa API neste caso estamos enviando o nome e senha usuário, e a API, por sua vez, encaminharia estes dados para o servidor, onde ele seria responsável por validar se o usuário que nós estamos buscando existe de fato, e caso exista, se a senha que nós enviamos na nossa request condiz com a senha que está armazenada no banco de dados. Após essas validações, o servidor envia, através da API, os dados da resposta, basicamente contendo o token, o nome do usuário, o e-mail e os demais dados que nós iremos receber. E, por sua vez, a API, no caso o garçom, encaminharia esses dados diretamente para o cliente, no caso o nosso aplicativo ou o nosso site. Espero que esta analogia tenha ajudado a vocês entenderem o que é uma API e a importância dela no dia a dia. Dentro deste contexto de API existem outros diversos conceitos que iremos abordar nas aulas futuras, como por exemplo verbos HTTP e status code. Uma coisa importante a ser lembrada é que este curso específico nós não iremos abordar o desenvolvimento de APIs específicas, por se tratar de uma trilha React Native.
