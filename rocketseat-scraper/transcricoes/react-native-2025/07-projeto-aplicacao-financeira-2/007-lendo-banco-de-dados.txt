WEBVTT E na aula de hoje nós vamos aprender a instalar e utilizar o BKeyper para vermos os dados que vão ser armazenados no banco de dados da nossa API. Nas aulas anteriores nós já aprendemos como clonar a nossa API, como colocar ela para rodar e também como ler as documentações. Porém é importante nós sabermos como nós olharmos os dados que estão sendo salvos no nosso banco de dados para validarmos se todos os dados estão sendo indevidamente criados ou buscados, vou estar disponibilizando o link do BKipper aí para vocês. Simplesmente clicar para fazer o download no sistema operacional que vocês estiverem utilizando, no meu caso aqui é o Mac, coloque o e-mail de vocês e o download vai inicializar automaticamente. E após realizarem a instalação, essa daqui é a cara do BKipper, ele é bem simples. Basicamente vocês irão clicar aqui em New Connection e Connection Type, vamos estar selecionando aqui SQLite, que é o banco de dados que eu utilizei para desenvolver o Backend, vamos estar selecionando aqui Choose File e agora vocês vão navegar até onde o projeto Backend de vocês está clonado, até onde vocês clonaram a API, vamos estar entrando aqui em SRC, Infra, Database, Type URMMoney, e vamos estar selecionando aqui o Database.sqlite muito importante que você já tenha rodado o comando de Migration Run, que eu ensinei pra vocês na aula que nós clonamos a nossa API que é ele que vai fazer a criação de fato desse arquivo, e vamos estar clicando aqui em Connect, e olha só, maravilha todas as nossas tabelas já estão aqui. Desta forma, a gente já consegue ter um controle de todos os usuários que estão cadastrados. Podemos ver aqui todas as transações que foram cadastradas também. Todas as Transaction Type, que no caso aqui é entrada e saída. Todas as Transaction Categories. E conforme nós vamos registrando novos usuários e cadastrando novas transações, todos os dados irão estar chegando aqui. E isso é bem interessante da gente ter um controle do que está sendo feito, do que está sendo salvo, se todos os dados estão consistentes com o que está aparecendo lá no nosso aplicativo.
