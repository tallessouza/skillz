Na aula de hoje criaremos um componente que será renderizado quando a nossa lista de transações estiver vazia para darmos um feedback para o usuário de que ele não possui nenhuma transação cadastrada. Para isso, aqui dentro da nossa page de home, vamos criar uma pasta nova chamada EmptyList. E aqui dentro vamos criar um arquivo chamado index.tsx e aqui dentro uma Functional Component padrão, Export Const Empty List. E aqui vamos simplesmente retornar um text importando do próprio React Native. Dentro desse text vamos colocar a mensagem de nenhuma transação encontrada. Agora vamos pegar o nosso componente de Empty List e voltar aqui na nossa page de ROM para dentro do parâmetro de list empty component, nosso list empty. Vamos importar ele. E agora para nós podermos ver o nosso componente de fato, vamos vir aqui dentro da propriedade de data, vamos remover o estado de transactions que está vindo lá do nosso contexto e vamos passar um array vazio. E olha só o nosso texto sendo renderizado na nossa flat list. Agora vamos aplicar um estilo aqui dentro do nosso text. Vamos colocar um class name contendo um text center, um text gray 600, um text lg para o tamanho da fonte, para ele ficar um pouco maior, e por fim, um margin top de 4. Na próxima aula, criaremos um componente responsável por mostrar para o usuário que o novo request está acontecendo, ou seja, vamos voltar aqui o nosso data para o nosso array de transactions. Quando chegarmos aqui no final da nossa lista, nós exibiremos um loading para o usuário, para que ele saiba que um request está acontecendo e que novas transações estão sendo buscadas. E um detalhe bem importante também aqui dentro do nosso list empty component é nós renderizarmos o nosso componente apenas quando o nosso initial loading for false. Então vamos simplesmente colocar uma condicional aqui. Se o nosso loading exponta initial for true, nós vamos simplesmente retornar um no caso contrário. Aí sim retornamos o nosso empty list. Dessa forma a gente impede com que a nossa mensagem de nenhuma transação encontrada seja exibida antes mesmo de realizar o request, até porque não faz sentido, nós não sabemos ainda se o usuário possui ou não transações enquanto esse request inicial não for concluído. Então, com isso, finalizamos a aula e espero vocês na próxima.
