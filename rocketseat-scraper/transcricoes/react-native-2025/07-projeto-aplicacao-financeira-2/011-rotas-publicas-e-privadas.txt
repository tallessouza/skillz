E aqui nós vamos utilizar um useCallback o useCallback basicamente é um método do react onde ele espera receber uma callback dentro dele e como segundo parâmetro ele espera receber um array de dependência assim como o useEffect agora o que esse useCallback faz? basicamente toda vez que um estado nosso no React é atualizado, todo componente é remontado. Inclusive todos os nossos métodos, todas as nossas funções. O useCallback vai fazer com que a função seja remontada novamente apenas quando alguma variável dentro do array de dependência for alterada. Como por exemplo, supondo que nós temos aqui a variável de user. Se o usuário realizar o login e o estado de user não estiver dentro da rede de dependência, o React não vai remontar esta função. Isso é bastante interessante porque pelo fato de nós estarmos lidando dentro dessa função com várias rotas, imagina o cenário em que toda e qualquer mudança de estado passa basicamente a remontar absolutamente toda a nossa aplicação. Isso não é nem um pouco performático, não é nem um pouco interessante. Ou seja, se o usuário não estiver logado, se o state de user for undefined, nós iremos simplesmente renderizar as nossas rotas públicas.
