WEBVTT Fala, Dev! Beleza? Na aula de hoje nós iremos entender o que é gerenciamento de estado global. Este é um conceito de suma importância que levará o nosso app para um outro nível. O gerenciamento de estado global vai permitir com que nossas aplicações fiquem muito mais escaláveis, mais organizadas e mais fáceis de dar manutenção. Então, bora entender o que significa isso na prática? O que é o gerenciamento de estado global? É basicamente uma forma de nós centralizarmos os dados que precisam ser usados por vários componentes de uma aplicação, facilitando o compartilhamento e a consistência dessas informações. Basicamente, até o momento, nós utilizamos apenas o useState para armazenar os estados localmente. Porém, isso tem algumas limitações, como por exemplo, imaginemos que teremos de exibir o nome do nosso usuário tanto no header da nossa aplicação quanto na nossa tela de perfil. Imagine que a gente precise passar todos os dados de usuário através de props em todos os níveis de aplicação até nós chegarmos onde a gente quer, que no caso seria no e na nossa tela de perfil. Isso seria muito ruim porque a gente acabaria deparando com um problema chamado que é quando nós passamos uma propriedade a vários níveis internos de componentes, do componente pai para o componente filho, para o componente avô e assim por diante. Então, em um cenário em que a nossa aplicação comece a crescer, comece a ter várias e várias telas diferentes, isso se torna um problema gigantesco, até porque o Prop Drilling torna o nosso código mais verboso e muito mais difícil de dar manutenção. Então, para isso, nós temos a solução, o gerenciamento de estado global, onde eu irei exemplificar para vocês de forma visual aqui no Miro. Basicamente nós temos aqui a nossa aplicação E basicamente na nossa aplicação nós temos a tela de login, a tela de home e a tela de perfil Supondo que após a realização da autenticação Nós vamos salvar todos os dados do usuário na nossa tela de login Isso acarretaria um problema em que para que a nossa tela de home e a nossa tela de perfil possuam acesso a esses dados A gente basicamente vai ter que criar aqui uma árvore de componentes muito grande Passando então os dados do usuário da tela de home através de props e da tela de home para a tela de perfil novamente através de props e dessa forma a gente criaria o nosso problema de prop drilling, nosso código iria ficar verboso e muito chato de fazer manutenção, então para resolver o nosso problema basicamente nós englobamos todas as nossas telas dentro de um provider. Esse provider é basicamente um contexto criado dentro da nossa aplicação onde podemos criar estados compartilhados entre os nossos componentes. Basicamente temos aqui o nosso provider e dentro do nosso provider temos basicamente uma Store onde é possível nós criarmos estados aqui dentro, onde esse estado será compartilhado com todos os outros componentes que estarão dentro do nosso provider. Basicamente dessa forma a gente garante que tanto a nossa tela de home quanto a nossa tela de perfil vai conseguir ter acesso a todos esses dados, ou seja, anteriormente os dados do usuário estavam dentro de login e nós passávamos através de props tanto para home quanto para perfil ou para quaisquer outros componentes que tenha necessidade de utilizar os dados do usuário, agora nós estamos simplesmente vindo aqui na nossa história simplificando o acesso desses dados e evitando sim o prep drilling. Existem diversas ferramentas no mercado onde a gente pode estar configurando o nosso estado global, como por exemplo o Redux ou o Zustand, onde nós iremos estar aprendendo o Zustand aqui no nosso curso mais à frente. Porém, neste nosso app nós iremos estar utilizando o Context API, que é a API nativa do React para a configuração de um estado global, de um contexto onde nós podemos compartilhar o estado com vários componentes de forma mais simples. Uma outra vantagem de nós estarmos utilizando um gerenciador de estado global como o Context API é a centralização de boa parte da lógica e regras de negócio dentro deles. Dessa forma, a gente garante que os nossos componentes fiquem mais simples, fiquem mais legíveis e fáceis de dar manutenção. Essa é uma outra vantagem que o gerenciador de estado global dá para a gente que, se nós soubermos utilizar da forma correta, acaba adiantando bastante a nossa vida no desenvolvimento, até mesmo na hora de dar manutenção em algum código.
