WEBVTT E na aula de hoje nós vamos aprender o que são status codes. Nada mais são do que pequenos códigos numéricos que dizem o que aconteceu com a nossa requisição. É de extrema importância nós entendermos o que cada um é, para que possamos tomar algum tipo de decisão no nosso app ou no nosso site. Bem, e os status codes são respostas numéricas enviadas pelo servidor sempre que um cliente, como um navegador ou app, faz uma requisição HTTP. Eles fazem uma requisição HTTP. Eles servem para indicar o resultado dessa requisição, ou seja, se deu certo, errado, se precisa de autenticação ou se o servidor está com problemas internos. Dito isso, nós temos três categorias principais de status codes, que é basicamente os status codes de sucesso, que são na família do número 200, os de erro do cliente, que são na família dos 400, e os erros de servidor, que são na família dos 500. 400 e os erros de servidor que são na família dos 500 e temos aqui alguns exemplos de status codes de sucesso como por exemplo o status code 200, que é a requisição bem sucedida geralmente é a mais genérica que tem ela não quer dizer que um tipo específico de ação foi realizada, normalmente quando a gente desenvolve uma API e não especifica o status code da resposta geralmente 200 é o padrão 201, created, recurso criado com sucesso, basicamente quando a gente chama alguma rota do tipo post, quando a gente cria algum dado através da nossa API, a E o 404, not found, que é basicamente de recurso não encontrado. Um exemplo bem comum deste caso é basicamente quando a gente tenta fazer uma autenticação de usuário e a gente simplesmente envia um e-mail que não está cadastrado no nosso banco de dados. Então, basicamente, a API vai retornar para a gente o not found, já que o usuário não foi encontrado. E, por fim, temos o status code de erro de servidor. Basicamente, o erro 500 é um erro genérico no servidor. Basicamente quando ocorre algum erro inesperado dentro do servidor ao tentar processar uma requisição, pode ser por bug no código ou falha analógica ou alguma exceção não tratada na nossa API. O 502, bad gateway, gateway ou proxy recebeu uma resposta inválida. O servidor recebeu uma resposta inválida de outro servidor, geralmente acontece quando o servidor funciona como proxy ou gateway, e algo deu errado com o serviço intermediário, exemplo, a API de gateway tentando se comunicar com o microserviço que está offline e acaba retornando esse erro. 503 Service Unavailable Este erro quer dizer que o servidor está temporariamente indisponível, simplesmente por alguma manutenção, como por exemplo, muitas vezes algumas empresas grandes anunciam que em certo período, em determinada data, em uma determinada hora do dia, os servidores estarão offline para manutenção. Então, geralmente, 503 é o erro que é retornado para a gente. Existem muito mais status codes para serem estudados, porém, esses daqui geralmente são os que vocês mais irão encontrar.
