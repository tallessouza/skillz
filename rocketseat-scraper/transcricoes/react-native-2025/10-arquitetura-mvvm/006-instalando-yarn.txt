WEBVTT Na aula de hoje, nós iremos configurar o YAR no nosso ambiente de desenvolvimento. Nós não vamos mais utilizar o NPM como gerenciador de pacote aqui da nossa aplicação, pelo simples fato de que estamos utilizando o SDK 54 do Expo e, por ser muito recente, pode ser que existam algumas bibliotecas incompatíveis, o que simplesmente gera algum tipo de conflito no momento da instalação. Então, para evitar dores de cabeça e quaisquer transtornos que possam acontecer, vamos utilizar o Yarn, pelo simples fato de que ele consegue lidar melhor com esse tipo de incompatibilidade do que o NPM. Vamos ir aqui no nosso navegador e vamos procurar o Yarn NPM. Vamos entrar aqui na documentação e vamos ver como é realizada a instalação. Vamos clicar aqui em Installation Guide e show aqui basicamente é rodar o npm install --global-yarn. Vamos voltar lá no nosso terminal. Como eu já tenho instalado, ele vai retornar este erro para mim, mas agora uma dica que eu dou para vocês é após a instalação do yarn, criem um terminal novo para garantir que tudo vai estar correto. Agora vamos apagar aqui a nossa package lockjson. Lembrando que é para excluir a package lockjson e não a package json. A PKGlock tem o intuito de travar todas as versões das nossas bibliotecas para garantir uma consistência em todas as libs, sempre que for necessário a instalação de alguma coisa. E vamos rodar simplesmente o comando de yarn. Dessa forma ele vai criar um arquivinho chamado yarn.lock que no fim do dia ele tem o mesmo propósito ali do PKGJSON. E show de bola, nossa yarn.loc já foi instalada com sucesso, agora podemos dar segmento aqui com o nosso curso naturalmente. Novamente, esta abordagem é justamente para evitar a dor de cabeça na hora de realizar a instalação de alguma biblioteca, pelo fato da SDK54 do Xposed ser muito recente, pode ser que haja alguma biblioteca que dê algum tipo de conflito, então para evitar quaisquer transtornos, vamos seguir utilizando o Yarn.
