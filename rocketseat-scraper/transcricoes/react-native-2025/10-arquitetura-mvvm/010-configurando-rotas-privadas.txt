E também nós temos aqui o nosso grupo de rotas privadas, no caso, rotas privadas, onde geralmente são a tela de home, talvez uma tela de configuração, talvez um dashboard, ou seja lá, ou algum tipo de rota que possua algum tipo de dado sensível, ou que talvez simplesmente seja necessário o usuário ter algum tipo de permissão específica para acessá-la. Nós temos esses dois grupos de rotas, e para nós realizarmos o gerenciamento de que em qual momento o usuário vai poder acessar uma ou a outra, nós vamos ter aqui basicamente uma tela de loading, que no caso será a nossa index, a index que nós temos aqui, onde na aula passada nós implementamos simplesmente o redirect para a tela de login futuramente nós iremos realizar esse tipo de validação que é basicamente a nossa tela irá acessar a nossa store do Zustand, no caso teremos aqui store Zustand, o nosso gerenciador de estado global onde todos os dados do usuário como o token também ficarão salvos ele irá realizar um request HTTP para o nosso servidor, ou seja, para a nossa API, o nosso back-end, enviando os dados que estão aqui dentro do Zustand, onde a própria API irá retornar para a tela de loading se a sessão do usuário está válida ou não, se o token ainda é válido e se de fato o usuário existe. E independendo da resposta do servidor, ou seja, se for uma resposta de sucesso, nós iremos redirecionar o nosso usuário para a nossa rota privada, ou seja, a nossa home, Para ele de fato poder ter acesso ao Marketplace, agora caso o servidor retorne pra gente um erro 500 ou um erro 400, nós vamos simplesmente redirecionar o usuário para a tela de login e de registro, pois muito provavelmente a sessão do usuário já foi finalizada, o Token provavelmente já deve ter expirado ou simplesmente o usuário não existe mais, e consequentemente realizaremos o redirect deste usuário para as rotas públicas para ele realizar o login novamente ou cadastrar um novo usuário.
