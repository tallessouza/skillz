O que o React Query fará, basicamente, é criar uma camada em volta do nosso Axios, criando um cache onde, em algumas requisições, em alguns casos específicos, nós consumiremos desse cache ao invés da requisição HTTP. Vou exemplificar de forma gráfica aqui para vocês no Scaly Draw. Em suma, nós temos aqui o nosso aplicativo e o nosso app precisa se comunicar com o nosso back-end, no caso, a nossa API. E para realizar essa comunicação, nós utilizamos o Axios, que no caso é a biblioteca que utilizamos desde o início da nossa trilha. Porém, no decorrer desse módulo, vocês verão que existem muitas requisições que não faz sentido nós batermos na API o tempo todo. Alguns dados não ficam em constante mudança, como no nosso caso de Marketplace, os dados da nossa loja não vão ficar atualizando o tempo todo, não é o tempo todo que um item terá um nome editado, terá uma foto editada, terá um texto, uma descrição. Então, o React Query entra meio que como um middleware entre o Axios e a API. Em suma, o React Query cria como se fosse uma camada de cache dentro do nosso aplicativo e algumas requisições em que nós fazemos para a nossa API. Quando a nossa API retorna os dados pra gente, o React Query salva internamente dentro do cache Então, da próxima vez que nós realizarmos o mesmo request com os mesmos parâmetros pra nossa rota, ao invés de nós batermos lá e termos toda a questão da execução do código assíncrono, o nosso aplicativo simplesmente vem e consome diretamente do cache, o que diminui muito os loadings que temos em muitas aplicações. Agora nós não vamos estar mais realizando uma requisição assíncrona num servidor externo, nós vamos simplesmente pegar no cache, que vai estar guardadinho bonitinho aqui dentro do nosso aplicativo, Isso daqui melhora muito a experiência do usuário com o nosso aplicativo, tudo fica mais fluido, e isso somado com algumas outras estratégias do desenvolvimento web, até mesmo o mobile, como por exemplo o Optimistic Update, que é dar feedbacks sempre positivas para o usuário enquanto um processamento em background acontece. Melhora muito a qualidade de vida do nosso usuário e esse tipo de coisa que nós vamos aprender neste módulo. Mas para isso, vamos configurar o React Query aqui no nosso aplicativo.
