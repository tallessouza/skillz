Na aula de hoje vamos entender um pouco melhor o contexto aqui da nossa aplicação, o que vai ser necessário nós configurarmos antes de darmos início com o desenvolvimento do nosso app. Então, como eu disse na aula passada, nós vamos reaproveitar todo um aplicativo que nós desenvolvemos no módulo passado. Então, você que fez o módulo passado e já tem o aplicativo em mãos, não precisa se preocupar quanto a isso. Vai ter que se preocupar mais com a parte do back engine em si. Agora, você que pulou o módulo passado, uma coisa que eu definitivamente não recomendo, você terá de clonar dois repositórios. No caso, o repositório aqui do Marketplace é Push Notification e também aqui do Marketplace Push Notification Backend. E você aí do... Você que já fez o módulo anterior, por mais que você já possua o Backend clonado na sua máquina, você vai ter de clonar um outro Backend, onde aqui nós temos essas funcionalidades novas que eu mostrei na aula passada. E pra isso é bem simples. Primeiramente, vamos abrir aqui o nosso terminal. Vamos navegar aqui até o diretório para nós armazenarmos aqui os nossos projetos. Vou criar aqui uma pasta, mkdir marketplace-push-notification. Vamos navegar aqui para marketplace-push-notification. Vamos vir aqui no repositório, no caso aqui vamos pegar o nosso back-end. Vamos vir aqui em HTTPS, copiar. Coloca aqui um git clone na url que nós copiamos. Vamos vir aqui também no repositório do aplicativo, você que não tem o aplicativo já desenvolvido do módulo anterior, vamos em https, copiar, voltar no terminal, novamente dar um git clone. Agora vamos entender um pouco o que nós temos aqui dentro dos nossos repositórios. Vou navegar aqui, dar um ls, cd marketplace, push notification backend, vou abrir ele aqui no VS Code. Maravilha, temos aqui o projeto em mãos. Como vocês podem ver, a estrutura é muito parecida com o do módulo anterior. É basicamente a mesma arquitetura, é o mesmo projeto, porém com o adicional de que aqui dentro da nossa .env.example temos algumas credenciais a mais que nós teremos de criar lá no OneSignal, que é basicamente o OneSignalAppId e o OneSignalApiKey, onde nós vamos aprender a criar essas credenciais em uma próxima aula. Aqui nós vamos focar em deixar o back-end rodando, funcionando. Vamos simplesmente abrir aqui o nosso terminal. Vamos rodar aqui o npm o yarn, para instalar todas as dependências. Perfeito, com todas as nossas dependências instaladas, agora nós vamos simplesmente rodar o comando yarn migration 2.run. Se ficarem na dúvida, é só vocês abrirem aqui a package.json e virem aqui nos comandos de migration run e migration revert. Dessa forma, nós vamos simplesmente executar todos os códigos de migrações que temos aqui para criar o nosso banco de dados local. E agora, com nossas migrations rodadas, vamos simplesmente rodar o yarn dev Na primeira execução, pode ser que ele demore um pouquinho, porque aqui nós temos alguns seeders para popular os bancos de dados, ou seja, nós estamos injetando aqui vários usuários, vários produtos no nosso banco de dados, para quando nós abrimos o nosso app, nós já tenhamos vários produtos já criados, para que nós possamos desfrutar, utilizar o aplicativo sem muita dor de cabeça, já tendo vários feedbacks visuais legais ali, e perfeito! Como vocês podem ver, o aplicativo, o Backend, funciona perfeitamente sem as nossas credenciais. Porém, se nós tentarmos realizar a redução de algum preço de algum produto, ele não irá conseguir enviar uma Push Notification, logicamente, pelo fato de que nós vamos ter de configurar o One Signal. Mas, novamente, vamos deixar isso aqui para uma próxima aula. Este daqui é basicamente tudo que nós precisamos fazer no momento para deixarmos o nosso Backend funcionando. Novamente nós vamos simplesmente clonar, instalar as dependências e por fim rodar a Migration e depois o Yarn Dev onde ele já automaticamente roda aqui todas as todos os seeders e agora para verificar se está funcionando de fato vamos simplesmente acessar aqui o nosso local host 2001-docs onde nós vamos abrir aqui as documentações do nosso back-end onde temos aqui todas as rotas e vocês que fizeram as aulas do módulo passado vão se familiarizar bastante com tudo que nós temos aqui porém agora temos algumas rotas novas como por exemplo aqui as de favoritos onde nós buscamos os produtos favoritados de um usuário onde nós adicionamos os produtos favoritos e onde nós removemos ele também E também temos aqui uma nova rota do tipo Patch, que é uma rota simplesmente para atualizar o preço de um produto, onde novamente, toda vez que nós realizarmos a redução do preço de algum produto específico e caso algum usuário dentro do nosso banco de dados tenha favoritado aquele produto, nós vamos de fato emitir uma Pudge Notification informando que houve uma promoção ou simplesmente uma alteração de preço no produto em questão. Agora, a partir da próxima aula, nós vamos entender como startar o aplicativo para todos os alunos, para todos que pularam o módulo anterior e vão utilizar o repositório que eu irei estar disponibilizando. Então, é isso. Espero vocês na próxima aula.
