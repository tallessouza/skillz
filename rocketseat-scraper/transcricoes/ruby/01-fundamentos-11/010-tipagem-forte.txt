WEBVTT Vamos falar agora sobre tipagem forte. Ruby tem uma tipagem forte. Isso significa que nós não conseguimos misturar tipos de dados diferentes. Se eu tentar misturar, por exemplo, somar um tipo de dado que não é compatível com outro tipo de dado, a linguagem vai reclamar. Vamos dar uma olhadinha em como que isso funciona aqui dentro do nosso console, que já vamos ver como que funciona também. Olha só, eu vou fazer aqui uma variável chamada i, que é um, e s, que é oito. Eu já dei uma pequena dica do nosso console, vamos ver logo em seguida. Vou tentar tomar i com s. Ele fala, mas peraí, um inteiro, uma string não pode ser convertida para um inteiro. Essa é a tipagem forte de Ruby atuando. Ela não deixa nós fazermos aqui uma soma, algum tipo de operação com objetos de tipos diferentes. Vamos dar uma olhada agora como que em JavaScript isso vai funcionar. Olha só, aqui nós já temos o nosso código em JavaScript, ele já está até falando o que vai acontecer. Aqui em JavaScript, ele é feito uma concatenação de um inteiro com um string. Ele tem uma tipagem fraca, ele consegue fazer esse tipo de concatenação de uma forma que às vezes é desejável, às vezes não é desejável. Em Ruby, nós precisamos explicitamente, nós vamos ver mais tarde, converter os dois tipos de dados para o mesmo tipo. Vamos dar uma olhada aqui em Java. Já deixei o Vim mais bonitinho. Vamos dar uma olhada no que vai vir a isso. Ops, forte. A Java já reclamou, porque ele consegue fazer a magiquinha ali da concatenação de string, mas ele fala, olha, você vai ter uma uma perda de conversão aqui, de um double para um int. Então, o compilador já para nessa hora e não deixa prosseguir. Vamos comentar só o código aqui embaixo. Vamos remover o código. E vamos dar uma olhada no que ele vai fazer. Java tem tipagem forte também, mas ela tem a mágica da conversão da string. Olha lá. Então, aqui nós conseguimos fazer com que um tipo que seja inteiro seja somado com uma string. No caso, novamente, é uma concatenação como nós vimos aqui em JavaScript. Mas Ruby tem uma tipagem, vamos falar assim, extra forte, que não permite que o tipo de dados diferentes sejam ali mixados.
