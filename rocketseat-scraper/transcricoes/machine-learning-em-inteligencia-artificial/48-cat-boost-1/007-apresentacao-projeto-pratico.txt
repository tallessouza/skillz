WEBVTT Vou apresentar agora o nosso projeto prático para demonstrar o algoritmo de Cat Boost. Nesse projeto, a gente está simulando aqui uma empresa do segmento financeiro que deseja aperfeiçoar seu modelo de riscos para clientes, pessoas jurídicas ou empresas, de forma que possa conceder crédito para essas empresas com mais segurança e menos exposição a esses riscos. Esse é um problema, eu diria, bastante comum no segmento financeiro. Existem, inclusive, plataformas, softwares de mercado, especializados em motores de risco, onde você tem lá, às vezes, milhares de variáveis sobre a empresa, na forma como ela se relaciona com aquela instituição, com o mercado, para que você possa, com base nessas informações, calcular o risco ou os riscos dessa empresa para poder conceder crédito, para poder conceder um financiamento, enfim, para estabelecer a forma como aquele banco se relaciona com aquele cliente. Então é um problema bastante comum no mercado. Para aperfeiçoar esse modelo, que é o que essa empresa deseja, o time de ciência de dados vai trabalhar em uma abordagem capaz de prever a probabilidade de três riscos diferentes, crédito, compliance e mercado. Além de uma classificação binária para cada um dos três riscos. Então, aqui é um problema multi-label, ou seja, o cliente, a pessoa jurídica, ela pode ter risco de crédito, compliance e mercado ao mesmo tempo, ou ela pode ter só de crédito ou só de compliance ou uma combinação entre esses três. Então a ideia é prever a probabilidade de cada um deles e também a classificação binária 0 ou 1 para cada um deles. É importante também que a gente entenda quais as variáveis influenciam mais em cada um dos targets, que são no caso os riscos de forma global e de individual. Ou seja, se eu quiser saber para uma empresa específica, por que ele disse que essa empresa tem uma chance de risco? A gente tem condições de responder essa pergunta. Para isso, a gente vai contar com o dataset, com os dados das empresas, que vão ter dados demográficos, dados de rating, além de algumas informações financeiras de relacionamento dessas empresas, de cada uma dessas empresas, com a empresa do segmento financeiro que está rodando o modelo. Dada a complexidade do desafio, a quantidade de variáveis categóricas, a gente vai usar um ensemble cat boost para fazer essa predição das variáveis target, das probabilidades de cada target e das possíveis razões para isso, principalmente quando o risco aparecer como verdadeiro. Como vai ser a estrutura do projeto? A gente vai carregar os dados, fazer uma análise exploratória, treinar modelos e ir armazenando as simulações e alterando hiperparâmetros e visualizar os resultados para avaliar a capacidade do modelo em ter feito boas previsões. Então agora, dada essa explicação, vamos para o código.
