WEBVTT Fala, Dave. Vou falar agora de uma etapa que é muito importante e a gente vai trabalhar durante toda a nossa trilha quando a gente vai fazendo os nossos modelos de machine learning. Essa etapa é a engenharia de features. Através dessa etapa, a gente vai conseguir selecionar, extrair, transformar ou criar novas variáveis para os nossos modelos a partir dos dados brutos que a gente recebe. Uma boa engenharia de features vai tornar o nosso modelo mais preciso, mais eficiente e, com certeza, mais interpretável. É importante lembrar que a engenharia de features é um processo iterativo, ou seja, você começa geralmente com um conjunto de features e vai testando novas combinações de features para determinar a melhor configuração para o seu modelo. Trago aqui algumas das tarefas que a gente executa dentro de um grande processo de engenharia de features. A primeira delas é a seleção de features, ou seja, baseado num conjunto de features que você tem à sua disposição, você selecionar um subconjunto dessas features que funcionam melhor para aquele modelo. Existem algumas técnicas para fazer essa seleção. Por exemplo, a pontuação da importância da feature e também a matriz de correlação. Essas duas técnicas, além de outras, podem determinar quais features são as mais relevantes para o treinamento do seu modelo. Uma outra etapa, uma outra atividade que a gente faz durante a engenharia de features é a transformação. Então, a gente pode normalizar uma variável, por exemplo, a gente tem uma variável que é numérica, e ela tem um valor de preço e eu quero, por exemplo, colocá-la em um range de 0 a 1 ou de menos 1 a 1. Geralmente, alguns modelos exigem isso e a gente faz isso durante essa atividade de transformação. Um outro tipo de transformação muito comum é a gente codificar variáveis categóricas ou fazer alguma transformação matemática. Por exemplo, eu tenho uma variável categórica que tem lá o gênero da pessoa ou a cor de um carro, ou, eventualmente, uma categoria de produtos, e a gente transforma essa variável para uma variável numérica, que faça mais sentido para o modelo que a gente está trabalhando. Além disso, na transformação é que a gente vai tratar features que têm dados ausentes ou features que têm dados que não são válidos para o seu modelo. Uma outra atividade importante durante a engenharia de features é a criação de novas features. Vamos supor que eu tenha uma feature que tem uma quantidade e eu tenho uma outra feature que tem um preço. Muitas vezes, para o meu modelo, trabalhar com essas duas variáveis de forma independente não vai trazer o mesmo valor se eu, por exemplo, fizer um cálculo matemático multiplicando essas duas informações. Então, a gente pode criar features novas que talvez façam mais sentido para a gente no nosso modelo do que trabalhar com features de forma independente. E por último, extração de features. Nesse caso, a gente usa um próprio processo de Machine Learning para que a gente possa extrair features numa técnica chamada de redução de dimensionalidade. Então imagine que você tem um conjunto muito grande de variáveis de features para trabalhar, você pode aplicar um processo de redução de dimensionalidade que vai reduzir a quantidade de features E aí essa quantidade de features reduzida é a que vai entrar para o seu modelo trabalhar, tornando ele mais eficiente. Então, diferente de um processo de transformação, a extração, ela usa o machine learning para poder trazer essas features novas para que a gente possa trabalhar.
