WEBVTT Da mesma forma que nós fizemos o projeto prático no bagging, a gente vai usar o mesmo dataset para esse projeto de boosting de classificação, aquele problema da edtech que quer avaliar o seu fator de conversão de leads. Nesse caso, ao invés de usar o bagging, a gente vai usar o método Adaptative Boosting de Assemble. Por que a gente vai usar um Adaptative Boosting e não um gradiente? Porque a gente vai ver mais pra frente, em outros módulos, outras variações de boosting usando o conceito de gradiente. E aqui a gente vai explorar mais o Adaptative Boosting de Assemble para essa questão. A estrutura do projeto vai ser da seguinte forma, a gente vai carregar os dados, nesse caso a gente vai carregar os dados limpos, que a gente salvou no projeto anterior, inclusive a parte toda de pré-processadores, para a gente ganhar tempo, a gente não precisaria fazer o EDA de novo, fazer todo esse tratamento de novo que já foi feito no projeto anterior, e a gente vai carregar os dados, treinar os ensambos e visualizar os resultados através das métricas e através da questão da interpretabilidade, ou seja, avaliar ali quais foram as variáveis que mais deram peso, que mais influenciaram no resultado do modelo. Então é dessa forma que a gente vai trabalhar nesse projeto. Agora vamos para o código.
