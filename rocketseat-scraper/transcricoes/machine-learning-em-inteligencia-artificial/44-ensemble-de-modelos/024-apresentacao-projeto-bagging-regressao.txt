WEBVTT Após termos concluído o primeiro projeto prático desse curso de introdução a Ensembl com Site Learning, a gente fez primeiro a utilização do algoritmo do tipo Bagging, o Bootstrapping Aggregation, para um problema de classificação. E agora eu quero trazer um outro projeto prático para a gente fazer, agora usando o Bagging, mas para um problema de regressão. Qual o desafio que nós vamos ter aqui? Então, nesse caso aqui, a gente tem uma Healthtech, que realizou uma pesquisa com mais de 1.300 pessoas sobre os gastos de saúde que essas pessoas tiveram, os gastos com saúde, nos últimos dois anos. A ideia dessa pesquisa era de criar uma oferta mais assertiva, mais, vamos dizer, direto ao ponto de planos de saúde para pequenas e médias empresas. Dentre os componentes para formatar essa oferta, é importante conseguir estimar os gastos de saúde dos funcionários de uma possível empresa cliente que essa Healthtech pode abordar, com base nas características desses funcionários dessas empresas. Dessa forma, para apoiar o desenvolvimento dessa oferta, a gente vai trabalhar num algoritmo de regressão, que possa estimar os gastos de saúde de funcionários. Considerando os volumes de dados e as features que nós temos disponíveis, a gente vai adotar o método de bagging, de ensemble, mas aqui, nesse caso, usando o algoritmo supervisionado de regressão e não de classificação. Em termos de estrutura do projeto, ele vai seguir a estrutura similar ao que a gente fez num projeto de bagging e classificação. Nós vamos carregar os dados, vamos fazer uma análise exploratória, vamos preparar esses dados para o modelo, vamos treinar esse ensemble e vamos visualizar os dados, tanto em termos de métricas, como de interpretabilidade. Então, bora lá para o código.
