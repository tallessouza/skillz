WEBVTT Um aspecto muito importante para ser gerenciado em projetos que lidam com dados, principalmente quando a gente vai colocar esses dados a serviço de uma solução de inteligência artificial, é como lidar com os dados ausentes dos seus conjuntos de dados. Então a gente vai ver aqui primeiro conceitualmente quais são os tipos de dados ausentes, as categorias, e depois tecnicamente como detectar esses dados ausentes dentro do seu conjunto, como removê-los ou como imputar valores ausentes, ou imputar valores que substituam os valores que estão ausentes. conceitualmente falando a gente tem três categorias de dados faltantes primeira delas são os dados faltantes completamente ao caso onde o fato de que um certo valor está faltando não tem nada a ver com seu valor hipotético e com os valores de outras variáveis Uma segunda classe, categoria de tipos ausentes, são os dados faltantes ao acaso. Nesse caso, faltar dados aleatoriamente significa que a propensão para um ponto de dados estar ausente não está relacionada aos dados ausentes, mas está relacionada a alguns dos dados observados, os dados que estão presentes. Em terceiro lugar, a gente tem os dados pautantes não ao acaso. E aí, duas razões são possíveis para que esse valor ausente aconteça. A primeira delas, que ela depende do valor hipotético. E aqui eu vou trazer um exemplo. Vamos imaginar, no caso do valor hipotético, que pessoas com salários altos geralmente não querem revelar suas rendas em alguma pesquisa. Então, nesse caso, esse valor ausente está relacionado a pessoas com salários altos. Em outro caso, que é o caso de você ter um valor ausente dependendo do valor de alguma outra variável, vamos imaginar, numa mesma pesquisa, que as mulheres geralmente não querem revelar a idade. Então, nesse caso, o valor em falta é a variável idade, mas que é impactado pela variável gênero. Então, aqui os dois exemplos para esses dados faltantes, não ao acaso. E agora?
