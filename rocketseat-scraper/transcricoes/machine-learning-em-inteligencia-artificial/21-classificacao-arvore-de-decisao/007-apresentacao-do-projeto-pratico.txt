WEBVTT Vou falar agora do... Como será o nosso projeto prático para o algoritmo de árvores de decisão. Então, o objetivo aqui é a gente ajudar uma empresa fictícia de concessão de crédito, que é a concessão de crédito para pequenas empresas, e ela possui um catálogo dos seus clientes. Os clientes dessa empresa são clientes PJ, clientes pessoas jurídicas, ou seja, outras empresas. E a gente tem informações dessas empresas, como, por exemplo, a idade da empresa, quanto tempo de vida a empresa tem, qual o faturamento mensal, qual o nível de inovação que essa empresa tem, entre outras informações. E aí, para que essa empresa de concessão de crédito possa dar um atendimento mais apropriado para cada tipo de cliente, eles classificam os clientes em alguns segmentos. Então, eles chamam de segmento starter, e aqui está na ordem. O starter, o bronze, o silver e o gold. E aí, qual a ideia? De forma que seja possível classificar novos clientes que estão chegando, a gente vai construir um classificador que vai determinar o segmento desse cliente com base nas informações disponíveis sobre o mesmo. E aqui, qual vai ser a estrutura do nosso projeto? A gente vai carregar os dados, vai realizar um EDA, nós vamos preparar o dataset de treino e teste, mas nós vamos preparar ele de outra forma. A gente vai usar o conceito de validação cruzada. Nós vamos treinar esse modelo, vamos validar esse modelo e vamos usar aqui uma estratégia de tunar hiperparâmetros. Então a gente vai aprender como tunar, com uma biblioteca bastante conhecida para isso, alguns hiperparâmetros da nossa árvore de decisão. vamos analisar algumas métricas desse modelo, vamos salvar esse modelo e, no final, a gente vai entregar esse modelo em uma aplicação batch. Como expliquei lá no começo, nesse caso, para que serve essa aplicação? Imagina que eu quero fazer uma previsão, determinar um segmento, mas eu não quero fazer isso individualmente, numa tela ou numa API. Imagina que eu tenho um arquivo, que eu tenho uma lista de clientes novos que estão chegando para mim, e eu queira receber um arquivo de volta com esses clientes, com os dados desses clientes e também qual é o segmento que eles deveriam pertencer. Então, a gente vai fazer uma coisa diferente dos outros módulos que não é API, não é aplicação. A gente vai jogar um arquivo para dentro do algoritmo, o algoritmo vai ler esse arquivo, vai devolver a predição e gerar um arquivo novo para que você possa ter essa predição nesse arquivo novo. E a gente vai aprender várias coisas diferentes aqui. O EDA, a gente vai usar bibliotecas diferentes de gráficas para mostrar os dados, a gente vai usar hiperparâmetros, a gente vai fazer a validação cruzada diferente do que a gente tem feito anteriormente. Então, aqui vai ser um projeto prático com muitas novidades em relação ao que a gente fez anteriormente. Então, agora, vamos para o código.
