Nesse caso aqui, a gente tem uma área, imagina uma área de RH de uma empresa, que ela possui informações do tempo de casa e do nível profissional dos seus colaboradores, dos seus profissionais. Ela deseja entender que se esses fatores, eles influenciam no salário atual desse profissional. Então, para isso, a gente vai treinar um algoritmo de regressão polinomial, de forma a ser possível prever o salário do profissional, dado o tempo de casa e o nível do profissional. Em termos de processo, a parte inicial já vai mudar um pouco. Então, a gente vai carregar os dados, vamos realizar uma análise exploratória, só que a gente vai preparar o dataset não mais no conceito de split, treino e teste. A gente vai preparar esse dataset já usando o conceito de validação cruzada. Depois, a gente vai rodar, na verdade, esse processo de treinar o modelo e validar o modelo duas vezes. A gente vai fazer um teste para saber, bom, será que uma regressão linear se ajustaria bem aos dados desse nosso dataset? E aí, se a gente descobrir que não, a gente vai aí sim rodar uma regressão polinomial. Então, é interessante a gente mostrar que, muitas vezes, não é... Ideal, você atacar de cara um problema já usando uma regressão polinomial. Muitas vezes, você pode usar, inicialmente, uma regressão linear, checar as métricas, e aí sim, se as métricas estiverem te mostrando que aquele modelo não está bem ajustado aos dados, você partir para outros modelos. No nosso caso, a gente vai partir para o modelo de regressão polinomial. Eu trouxe aqui também uma outra validação que a gente vai fazer no modelo, que é a validação de overfitting. A gente quer evitar, especificamente no caso da regressão polinomial, que a gente tenha overfitting no nosso modelo. Ou seja, o nosso modelo tenha aprendido muito bem as características somente para o conjunto de treinamento, mas quando ele vai para o conjunto de testes, ele não consegue generalizar muito bem. Lembrando que aqui no nosso conjunto de testes, a gente vai ter um conjunto de testes que vai ser, por exemplo, 30% de todo o conjunto. Nós vamos ter, no modelo de validação cruzada, a gente vai splitar esses dados em uma série de conjuntos, ele vai sempre separar, ele vai fazer interações nesses conjuntos. Então, a gente vai, por exemplo, pegar cinco pastas, os cinco conjuntos, a gente vai dividir o nosso dataset em cinco partes. Para cada interação, ele vai pegar uma dessas partes e usar para teste e vai usar as outras quatro para treinamento. Ele vai fazendo iterações até que todas as partes, as cinco partes daquele dataset tenham sido usadas tanto para treinamento quanto para teste. E aí depois a gente tira uma média das métricas para a gente avaliar como aquele modelo foi. Depois de a gente ter feito isso, a gente salva o modelo. E aí também tem uma outra mudança.
