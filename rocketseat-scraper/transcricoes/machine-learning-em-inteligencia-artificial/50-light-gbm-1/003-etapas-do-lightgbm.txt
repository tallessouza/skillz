Primeiro passo, ele inicializa com uma previsão base, depois ele faz o cálculo dos resíduos ou dos erros com base nessa previsão, terceiro, ele cria uma árvore de decisão para ajustar os resíduos que foram calculados, depois ele atualiza as previsões com base nessa nova árvore que foi criada e vai repetir nesse processo até atingir um critério de parada. A gente vai ver a partir de agora cada uma dessas etapas do processo do LightGBM. Primeira delas, inicializar com a previsão base. O algoritmo começa com uma previsão inicial simples, que por uma regressão é geralmente a média dos valores do alvo, ou a probabilidade inicial para uma classificação binária. Essa previsão é usada como base para calcular os resíduos iniciais que representam o erro entre as previsões e os valores reais. Após essa inicialização, é feito o cálculo dos resíduos ou dos erros, que são a diferença entre os valores reais e as previsões atuais. Eles se tornam o ovo-alvo para as próximas árvores, pois o objetivo do algoritmo é ajustar essas diferenças. Após a criação, o cálculo desses resíduos, é criada uma árvore de decisão para que ajuste esses resíduos. E aí a gente tem vários steps dentro dessa árvore de decisão. O primeiro deles é que, para criar essa árvore, ele usa o método Leafwise, que eu expliquei anteriormente, que prioriza a expansão das folhas, que oferece o maior ganho de informação, fazendo com que o erro se reduza. No método Leafwise, a árvore cresce desbalanceada, expandindo seletivamente as folhas mais importantes. Um segundo aspecto que acontece aqui dentro da árvore de decisão que é criada é essa divisão por histograma. Então o LightGBM converte os valores das variáveis em bins, que são grupos ou faixas, construindo um histograma de frequência. Então ao invés de ele avaliar todas as possíveis divisões, o algoritmo analisa as divisões com base nesses bins, nessas faixas, reduzindo significativamente o custo computacional. Um outro aspecto que é feito dentro dessa árvore, como o algoritmo é um algoritmo de gradient boost, ele também ajusta os pesos das folhas para minimizar os ruídos. Esses ajustes são feitos com base no gradiente da função de perda, ponto esse que também foi explicado no módulo anterior de Cat Boost, que mede o impacto dos pesos no erro. O gradiente é uma indicação da direção e magnitude da mudança necessária para melhorar a previsão. Após essa árvore ser construída com esses três passos, é feita a atualização das previsões com base na nova árvore. Então, as previsões do modelo são atualizadas somando as contribuições da nova árvore às previsões anteriores. Essa atualização é controlada principalmente pelo hiperparâmetro chamado Learning Rate, que a gente também explicou no módulo anterior, que determina o impacto da nova árvore no modelo global que vai ser criado no final do processo. Esse processo de calcular os resíduos, criar uma árvore e ajustar, ele vai se repetindo até atingir um critério de parada. Esse critério de parada pode seguir algum dos tópicos que estão aqui abaixo. Primeiro, o número máximo de árvores, que é um hiperparâmetro chamado N-estimators. Ele interrompe o treinamento se o desempenho no conjunto de validação não melhorar após um número pré-definido de interações. Além desse processo de repetição, acontece durante o treinamento as técnicas de regularização que eu comentei também como uma das principais características do Light GBM.
