O MLflow é uma plataforma open source que foi criada pela Databricks, uma empresa gigante no mundo de dados e inteligência artificial, em 2018. E ela foi criada para gerenciar todo o ciclo de projetos de machine learning, desde o experimento até o deployment em produção. Ele foi desenvolvido para resolver três grandes desafios comuns no machine learning. O primeiro, o tracking de experimentos. O segundo, a reprodutibilidade de resultados e o deployment padronizado de modelos. Aqui no desenho eu mostro primeiro uma visão geral do workflow de Machine Learning e as pessoas envolvidas nesse workflow. Então, a gente percebe o seguinte, se a gente for olhar desde a preparação de dados até o monitoramento do modelo, nesses sete passos aqui, várias pessoas têm interesses ou são envolvidas nesses processos. Então, no caso do Data Preparation, a gente tem o Data Engineer, que tem ali o interesse de registrar corretamente os seus dados, os datasets que vão ser usados para o treinamento, Entre a fase de Exploratory Data Analysis, o EDA que a gente costuma fazer, Feature Engineering e o treinamento do modelo e a validação do modelo, geralmente isso aqui está ligado a uma figura de um cientista de dados. Especificamente na validação do modelo, a gente já começa a ver o que? o usuário de negócio, o stakeholder de negócio, ajudando a validar esse modelo pelo ponto de vista, não das métricas técnicas, mas das métricas de negócio. E a gente começa a ver também a figura do ML Engineer, que é, em geral, a pessoa que faz esse trabalho de MLOps. Feito esse trabalho de validação de modelo, a gente começa a partir para o modelo para as fases de deployment e monitoramento. onde eu continuo envolvendo o stakeholder de negócios e o ML Engineer que conclui esse processo de MLOps. Então, o MLflow, ele cuida, ele dá essa possibilidade de como a biblioteca Open Source cuidar de todo esse fluxo de Machine Learning. Olhando pelo ponto de vista do ciclo de vida de modelo no MLflow, especificamente no modelo, a gente tem um processo onde eu tenho um treinamento de modelo, Dunning de hiperparâmetros e avaliação. E eu vou registrando todo esse ciclo até que eu chegue no modelo que eu queira validar. Eu vou fazer a validação dele pelo ponto de vista do negócio. Esse modelo validado, eu registro esse modelo num registre dentro do MLflow, e aí esse modelo fica pronto para deployment. Então, essa aqui é uma visão geral do MLflow. Falando especificamente de controle de experimentos, o ML4D fornece um conjunto de ferramentas para cuidar do fluxo como um todo. Então aqui eu vou dividir quais são esses principais modos. Então eu tenho o módulo de tracking, que permite registrar, organizar e comparar os experimentos de ML, e a gente vai usar ele aqui, acompanhando parâmetros, métricas e artefatos. Então, esse é o foco do que a gente vai ver agora, que é o controle de experimentos. Eu tenho um outro módulo, um componente chamado Model Registry, onde eu faço todo o versionamento e a gestão de ciclo de vida do modelo. Então, eu posso colocar o modelo em staging, produção, archived, e ainda documento, reviso e promovo esses modelos. Eu tenho uma funcionalidade nova no MLflow, que é o Deployments for LLM, que oferece uma API para servir modelos de LLM de diversas fontes com autenticação e segurança integradas. E ele serve tanto para modelos SaaS quanto para modelos Open Source. Eu tenho um outro módulo, que é o módulo de Evaluate, que permite automatizar a avaliação, comparação e validação de modelos tradicionais e também de LLMs. Eles facilitam as decisões baseadas em dados para a melhoria contínua dos modelos que eu estou colocando em produção. Outra funcionalidade nova que chegou com o advento da IA generativa é o Prompt Engineer UI, que é uma interface que permite testar, ajustar e validar prompts para modelos de linguagem. Você consegue comparar o resultado e realizar deploy diretamente da plataforma. Ele acaba acelerando a experimentação e refinamento para aplicações com LLMs. Por último, eu tenho um módulo de projetos que permite o empacotamento do código das dependências dos scripts de execução de machine learning em projetos portáteis. Ele usa descritivos para garantir a reprodutibilidade em diferentes ambientes. Ele também suporta a execução local ou remota de experimentos. Por fim, dando um enfoque especificamente na parte de controle de experimentos com o módulo Tracking, o que a gente vai fazer aqui é usar uma instância Localhost, onde a gente vai ter um código Machine Learning. Nesse código, a gente vai invocar a API de Tracking dentro desse nosso código, nosso Jupyter Notebook, onde a gente vai poder colocar metas dados, artefatos para um arquivo local. Esse arquivo local vai ser acessado através de uma UI, que eu vou mostrar como que a gente acessa, e você vai ter todo um panel, um dashboard, comparando todas as informações que você logou. O módulo Tracking, ele permite, como comentei, acompanhar parâmetros, métricas, versões de código, artefatos, inclusive arquivos de modelo, imagens e logs, durante as execuções dos experimentos de Machine Learning.
