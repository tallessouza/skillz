Qual que é o escopo desse trabalho? Uma empresa global que provee um buscador inteligente de passagens aéreas tem sido alvo de críticas do seu algoritmo de busca e precisa recuperar a sua reputação em sua principal fonte de receita. Imagina um buscador tipo o Skyscanner, um kayak, dentre outros. Só que é um buscador que não está buscando algo realmente de valor para os seus usuários. Então para isso, para ele recuperar essa reputação, ele pretende refinar seu modelo de recomendação, não apenas trazendo as passagens conforme o argumento de busca, mas indicando qual a melhor decisão que o cliente deve tomar para cada opção de passagem retornada na busca. Como esse buscador, que é conveniado a diversas companhias aéreas, recebe o feedback do cliente em relação à compra ou não de passagens indicadas pelo buscador, foi possível consolidar uma base de mais de 18 mil passagens, com a decisão final do cliente sobre cada passagem. Nosso objetivo é criar um modelo, usando uma rede neural profunda, que consiga classificar a passagem contra a decisão que o cliente deveria tomar. Muito mais no detalhe quando a gente for mostrar o dataset. Qual que é a estrutura da base de passagens? Nessa base eu tenho um localizador, que é aquelas seis caracteres, letras e números que identificam de forma única uma passagem, um ticket, qual a companhia aérea, qual a origem daquela passagem, o aeroporto, qual o destino, o aeroporto, qual a classe dessa passagem, quantas paradas esse voo vai ter ou teve, no caso aqui é uma base já histórica, qual o tipo de destino, qual a estação do ano que foi essa viagem, qual o preço atual da passagem, quando foi comprada, quando a pessoa escolheu, qual o preço da passagem na semana anterior e qual o preço da passagem no mês anterior. Com base nesses argumentos, nessas features dessa base de dados, a gente vai ter uma variável target, que é a decisão do cliente. A gente vai fazer uma sugestão de decisão para o cliente. Vocês vão ver com mais detalhes quando a gente for fazer a exploração dos dados. Como que vai ser o nosso fluxo de trabalho? A gente vai carregar os dados, fazer uma análise exploratória, preparar os dados para treinamento, treinar essa rede neural profunda, visualizar os resultados através da parte de experimentos e métricas do MLflow, Podemos fazer outros treinamentos, a ideia é que a gente mostrar vários experimentos em métricas do MLflow e visualmente como você consegue enxergar isso mudando hiperparâmetros do seu modelo e por fim fazer o deploy desse modelo dentro do MLflow.
