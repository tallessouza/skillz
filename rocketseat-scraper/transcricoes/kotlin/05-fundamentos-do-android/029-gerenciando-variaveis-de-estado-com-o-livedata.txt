WEBVTT Nessa aula vamos aprender como utilizar o LiveData para poder emitir tipos de dados observáveis no Android. Bom, o primeiro LiveData foi criado exatamente para isso, para gerar tipos de dados que são observáveis, e ele vai permitir que as views possam observar as mudanças dos dados e fazer com que a interface gráfica possa ser atualizada automaticamente. Então, dentro de um LiveData, a gente vai ter um valor que vai poder ser observado via Observers. O padrão dos LiveDatas é gerar essa observabilidade através dos Observers, normalmente vinculados a Activities ou Fragments. E quando o valor do LiveData for alterado, todos os observers que estiverem registrados vão ser notificados com esse novo valor que foi recebido dentro do LiveData. E se eles estiverem com o estado ativo, ou seja, uma Activity ou um Fragment estiver visível, eles vão atualizar a UI de forma totalmente automática. E além disso, o LiveData é considerado Lifecycle Aware. Ele respeita o ciclo de vida dos componentes observadores. Então, um Activity, um Fragment, quando ele vai para o Background ou quando é destruído, automaticamente ele efetua o corte da observação feita dentro daquele dado. O estado inativo vai fazer com que um LiveData não notifique esses observadores que não estão ativos. Então vamos lá.
