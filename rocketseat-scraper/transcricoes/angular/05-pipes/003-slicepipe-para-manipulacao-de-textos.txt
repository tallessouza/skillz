Ele já vem com o padrão que implementa a funcionalidade do método Array.prototype.slice e String.prototype.slice do JavaScript. Então ele simula ali o Slice nativo ali do JavaScript, beleza? Seu uso principal é criar uma subsessão de uma coleção de Array ou de String. Então, extrair uma parte de um String ou extrair uma parte de um Array. Basicamente é para isso que ele serve. Então, como que nós utilizamos ele? Nós vamos passar um valor o sinalzinho de pipe, o nome dele que é slice, dois pontos o parâmetro start, dois pontos o parâmetro end. O parâmetro end ele é opcional, tá bom? Esses coxetes aqui é só para indicar que ele é opcional. O value é o dado de entrada, é a nossa fonte de dados, que pode ser uma string ou um array, que eu quero extrair algo dessa fonte. O start é o parâmetro obrigatório do pipe, que define o ponto de partida dessa extração. E o end é o parâmetro que é opcional e que define o ponto de parada dessa extração, até onde eu vou na extração desse value. Bom, aqui é o seguinte, o Start e o End vão receber valores, números, indicados ali pelo índice 0, 1, 2, 3, começando sempre do zero, beleza? Só que tem umas diferenças aqui no que é retornado em cada um deles, principalmente aqui no End. Então, se eu passar "-1", eu vou estar começando do final daquele array ou daquela string a extrair ali os valores, beleza? Caso eu passe um valor que é maior que o tamanho de caracteres de uma String ou maior do que o tamanho de itens de um Array, ele vai retornar para mim um Array vazio ou uma String vazia. Já no parâmetro end, se eu omitir ele, a extração vai do índice start até o final da String ou do Array. Então ele vai começar do parâmetro start, que é obrigatório, e vai até o final, pegando tudo, beleza? Todos os itens ou todos os caracteres até chegar no final.
