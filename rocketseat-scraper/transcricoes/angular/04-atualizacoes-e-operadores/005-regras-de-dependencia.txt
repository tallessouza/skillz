O que é uma dependência? Por exemplo, um componente consumir um outro componente. um componente consumir uma service, uma service consumir uma outra service, então acaba se criando uma dependência entre um e outro, certo? Só que tem umas regrinhas aqui, olha só. Regras de dependência. Vamos ter a camada, que é a Feature, Shared e Core. Pode depender de e não pode depender de. Então, uma Feature, ela pode depender da Core e da Shared. Então, uma Feature ali, o componente dentro de uma Feature, pode consumir um service da Core ou um service da Shared. um componente da core um componente da shared isso é normal acontecer beleza ela não pode depender de outras features beleza então a sua feature ela tem que ser isolada ali no contexto dela pode ser acontece tá de você precisar consumir um componente de uma outra feature até e tranquila mas a sua componente da feature que consome no caso por exemplo aqui ó de colocar algo mais visual para gente ver Por exemplo, vamos supor que o componente dashboard consome, por algum motivo, o componente user list, para mostrar ali uma lista de usuários. Isso pode acontecer? Pode acontecer, é normal também. Porém... Esse componente dashboard não pode depender desse componente para funcionar. Se eu remover esse componente por algum motivo, user list do dashboard, removi ele, removi o seletor lá, esse componente dashboard tem que funcionar normalmente, certo? Sempre pense nisso. O ideal seria esse componente ser totalmente isolado e consumir apenas os seus componentes internos. Mas no dia a dia, em aplicações complexas, isso é muito difícil de acontecer. Então é normal você acabar consumindo um componente de uma outra feature. Mas lembra dessa regra. Ele, se for removido, não vai quebrar o componente consumidor. Beleza? Vamos voltar lá.
