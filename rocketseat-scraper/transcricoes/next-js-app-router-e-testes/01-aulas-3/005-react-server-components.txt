Só que, qual que é a grande diferença então do Client Component? A grande diferença do Client Component é que este componente que possui essa anotação useClient, ele vai passar por um processo no Next que a gente chama de hidratação. Hidratação e unhas, não, brincadeira. Hidratação. Hidratação é o quê? É um processo que já é muito uma convenção que existe na comunidade JavaScript. Hidratação significa pegar um HTML que não possui qualquer tipo de interatividade, um HTML cru, que não tem JavaScript, é só HTML seco, sem qualquer tipo de interatividade, qualquer tipo de dinamismo, e converter ele em um HTML interativo. Como a gente faz isso? Adicionando uma camada de JavaScript dentro dele. Ou seja, hidratação é o ato de pegar um HTML seco e converter ele em um HTML interativo com JavaScript. Isso aqui o Next faz em todos os componentes que possuem a diretiva useClient. Então, quando a gente sinaliza que um componente é um client component, quer dizer que este componente vai ser hidratado. Ou seja, todo o JavaScript necessário para que esse componente seja exibido em tela vai ser enviado para o navegador. E aí, é como se esse componente... Como não? É o que realmente acontece. Esse componente é renderizado no Node, E aí, quando ele chega para o usuário, lá no navegador, o JavaScript necessário para esse componente funcionar é carregado, e é claro que é um JavaScript bem menor do que aquele 5 MB que a gente tinha anteriormente, e aí esse componente, digamos, é atualizado ali no navegador.
