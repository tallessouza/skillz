WEBVTT Nessa aula, a gente vai começar a configurar e utilizar uma ferramenta que eu acho fantástica, que é o Drizzle. O Drizzle é um ORM para JavaScript, assim como Prisma, assim como TypeORM, SQLite. Só que a grande sacada do Drizzle é que, novamente, como a gente falou sobre o Rhono e o Elysia, o Drizzle não utiliza nenhuma API específica do Node, ou seja, ele é um ORM agnóstico de Runtime. Ele pode ser executado em qualquer ambiente que executa JavaScript. Ou seja, a gente consegue usar o Drizzle tanto no Node quanto no BAN, no DIN, nos Cloudflare Workers, na Versa. O Drizzle é totalmente agnóstico. Outro grande diferencial do Drizzle é que ele não cria uma ponte entre a sua aplicação e o banco de dados. O que eu quero dizer com isso? No Prisma, quando a gente faz qualquer tipo de operação no banco de dados, essa operação não vai direto para o banco de dados. A nossa aplicação envia para um binário que é feito em Rust, criado pelo time do Prisma. Esse binário é criado pelo time do Prisma. ele interpreta a nossa query e envia ela para o banco de dados, ou seja, existe uma ponte ali no meio que está fazendo o envio, o repasse da comunicação entre a nossa aplicação e o banco de dados. Isso, na grande maioria das vezes, não implica em nada para 99% das aplicações, Mas em casos de uso mais complexos, isso pode causar alguns problemas de performance, porque o Prisma acaba fazendo algumas coisas automáticas e executando algumas queries automáticas que podem gerar alguns efeitos colaterais no futuro, caso a sua aplicação cresça muito, muito, muito, muito. Mas o Drizzle não faz isso, ele é basicamente um Query Builder. Então, apesar de ele ser um ORM, e aqui eles usam a sigla ORM de uma maneira um pouco errada, porque eles usam o ORM, na verdade, como uma ferramenta que tem todo o funcional, todo o ferramental para um banco de dados. Então, não é só simplesmente a parte de Query, mas toda a parte de Migrations, toda a parte do estúdio para visualizar o banco de dados. Mas, na verdade, o Drizzle segue uma API muito mais próxima de um Query Builder ao invés de um ARM, onde a gente escreve as nossas queries assim como é muito próximo do que é um SQL da vida. Isso, na minha opinião, é outra vantagem até, de certa forma. É uma desvantagem porque as queries ficam um pouquinho mais verbosas, mas ao mesmo tempo você não perde o seu conhecimento em SQL. Você consegue aproveitar todo o seu conhecimento em SQL na hora de escrever código com o Drizzle. Mas lembrando, é uma alternativa, não necessariamente é a melhor. O Prisma com certeza tem uma experiência de desenvolvimento melhor que o Drizzle, mas como a gente já tem conteúdo com o Prisma aqui dentro, resolvi trazer uma ferramenta nova. para você conhecer outras ferramentas.
