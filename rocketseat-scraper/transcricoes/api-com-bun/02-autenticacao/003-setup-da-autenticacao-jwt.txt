No JWT existem vários... Algoritmos para criar um token JWT, o mais comum que é o padrão que ele vem é o HS256. Se eu não me engano o HS256 é um algoritmo onde a gente tem uma chave secreta. Essa chave secreta que é uma string aleatória, digamos assim. Ela é usada para fazer uma assinatura no token, que não é um hash, é uma assinatura. A assinatura é uma forma da gente, assim como na vida real, a gente saber que aquele token foi gerado através de uma entidade que a gente confia, que no caso é o nosso próprio back-end. Ou seja, o nosso back-end guarda uma informação, que é um segredo, E ele cria todos os tokens assinados utilizando esse segredo. E assim, em algum momento, se eu precisar saber se esse token foi assinado pelo meu back-end ou não, eu preciso saber do segredo. Ou seja, é o algoritmo mais comum utilizado para autenticação Principalmente quando eu estou criando um monolito, todo o back-end da minha aplicação em um lugar só. Mas existem outros algoritmos, por exemplo, o RS-256 ou o RS-512, onde eu tenho uma chave privada e uma chave pública. E aí já é um pouquinho diferente, é um conceito de chaves assimétricas, onde eu crio os tokens, eu faço a assinatura do token utilizando uma chave privada e eu posso validar se aquele token foi criado pelo meu back-end através de uma chave pública. E como o próprio nome já diz, chave pública pode ser compartilhada internet afora porque ela não pode ser utilizada para a criação de novos tokens. Ou seja, a chave privada, apenas o meu back-end que cria novos tokens vai conhecer, E a chave pública pode ser compartilhada com qualquer sistema que queira averiguar, garantir que a identidade daquele token é real, foi gerada pela minha chave privada.
