O NPM é o Node Package Manager, ou seja, o gerenciador de pacotes padrão do Node. A gente utiliza o NPM para fazer a instalação de pacotes na nossa aplicação e instalar dependências. E a gente utilizou, por exemplo, para fazer aqui a instalação do próprio TypeScript. A gente utilizou NPM, espaço e abreviação de install. Instalamos, por exemplo, o TypeScript e a gente separa por espaço para instalar em uma mesma instalação, digamos, mais pacotes como a gente instalou aqui, por exemplo, o 2. Além do TypeScript, a gente também instalou o arroba types barra node aqui. E aqui a gente especificou a versão que a gente está utilizando. Então, muito bem, o NPM a gente utiliza para instalar, beleza? Pacotes. Agora, o NPX é conhecido como um package running, ou seja, ele executa pacotes. O significado dele é Node Package Execute, ou seja, ele executa pacotes. Mas, como eu disse, também é conhecido por rodar pacotes, o package running, enfim. E o legal do NPX é que ele executa binários dos pacotes, que são as dependências que a gente tem disponível dentro da nossa aplicação. Por exemplo, a gente tem aqui dentro do nosso Node Underline Module. Está vendo que tem essa pasta .bin? Aqui a gente tem os binários das bibliotecas que a gente tem aqui no nosso projeto. Está vendo que a gente tem aqui, por exemplo, o .tsc. Quando a gente executa com o NPX .tsc e coloca aqui o arquivo que a gente quer fazer a compilação, perceba que aqui o NPX é como se fosse um atalho dizendo que a gente quer executar esse .tsc aqui, esse binário aqui, que vai, então, converter o nosso arquivo de TypeScript para JavaScript.
