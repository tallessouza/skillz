WEBVTT A partir de agora, vamos falar sobre armazenamento de dados. Existem estratégias para salvar dados, você pode salvar os dados em memory, ou seja, na própria memória, só que tem uma desvantagem, se o seu servidor, se a sua aplicação, ela para de executar e você executa ela novamente, os dados se perdem, porque a memória ela é volátil. Então, os dados, quando salvos na memória, eles não são salvos ou armazenados de forma permanente, então essa é a desvantagem, você pode perder os dados. O in-memory é mais utilizado quando a gente quer fazer testes ou quando é uma informação que a gente não precisa guardar de forma permanente, tá? Muito bem, temos agora como estratégia para guardar os dados de forma permanente, arquivo, então a gente pode criar, por exemplo, um arquivo do tipo JSON e armazenar, utilizando a própria estrutura JSON dentro desse arquivo os dados, criando, por exemplo, uma lista de objetos e aí cada objeto tem os dados ali, as informações que a gente quer armazenar, e também podemos utilizar a estratégia de banco de dados, é uma outra alternativa para armazenar os dados de forma permanente, ou seja, mesmo que a sua aplicação pare de executar e você execute ela novamente, os dados, quando estão armazenados em um arquivo ou em um banco de dados, por exemplo, os dados ficam armazenados de forma permanente e a sua aplicação consegue ler o conteúdo, seja do arquivo ou do banco de dados para exibir ele ali dentro da sua aplicação, para o usuário e assim por diante, tá?
