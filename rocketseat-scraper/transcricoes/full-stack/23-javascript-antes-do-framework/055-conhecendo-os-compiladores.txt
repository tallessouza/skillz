Bom, basicamente o objetivo do compilador e porque a gente utiliza e vai utilizar compilador no javascript é porque ele converte o código que foi escrito em uma determinada versão para outra versão e aí talvez você esteja se perguntando qual é a vantagem disso e por que fazer isso, por que utilizar o compilador para realizar essa tarefa. Porque aí a gente consegue utilizar os recursos mais recentes da linguagem mesmo em ambientes que não oferecem suporte para essas funcionalidades e o clássico exemplo é navegadores mais antigo. Os navegadores mais antigos e que utilizam versões anteriores do javascript não tem os recursos mais modernos, por isso a gente precisa manter uma compatibilidade com o máximo de navegadores possível para que a nossa aplicação esteja disponível para o maior número de usuários e aí que entra então a vantagem de utilizar o compilador para pegar o código que a gente escreveu com as funcionalidades mais atuais do javascript e converter para que o nosso código final seja compatível com esses navegadores que também são mais antigos. Então esse é o objetivo do compilador. Existe dentro disso a transpilação. A transpilação também é um termo utilizado para se referir à compilação para versões anteriores e dentro desse fluxo de transpilação basicamente a gente tem três etapas. As coisas acontecem basicamente em três fases. A primeira de parser, que é o mapeamento de todos os elementos do código javascript. Então mapeia tudo, tudo aquilo que a gente tem no nosso código. Na segunda etapa, que é a etapa de transformer, a gente tem a manipulação desses elementos para poder gerar o código compatível que é feito na terceira etapa de generator, que é onde a gente vai ter de fato o código compatível sendo gerado ali para ser então utilizado para ter suporte para os navegadores que utilizam versões anteriores do javascript. Imagina só se você tivesse que fazer versão por versão, a gente ficaria maluco e também humanamente impossível praticamente ficar replicando o código para dar suporte para todos os navegadores, incluindo esses que usam versões anteriores. Por isso então entra essa motivação de utilizar os compiladores para cuidar desse processo para a gente automatizar isso. Desse jeito utilizando essa ferramenta, ferramentas de compilação, a gente foca em utilizar o que tem de mais novo do JavaScript e deixa que o compilador se encarregue de oferecer suporte, de converter o nosso código para uma versão que vai dar suporte para o maior número de navegadores. Então, falando dessa motivação, os compiladores e esse processo de transpilação é importante até para a evolução da própria linguagem, porque a gente pode utilizar as novidades da linguagem mesmo em ambientes que não oferece suporte para elas.
