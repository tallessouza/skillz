WEBVTT Nessa aula eu vou mostrar pra você como utilizar o webpack dev server. A vantagem de utilizar ele ali junto com o webpack é que a gente consegue subir um servidorzinho local pra rodar a nossa aplicação. Igual a gente faz aqui quando, por exemplo, eu venho e utilizo no index.html o live server. Ele abre um servidorzinho e aí se a gente modifica o arquivo ali isso vai refletir e tudo mais. A gente pode configurar pra isso acontecer. Então vamos utilizar o webpack dev server e aí a gente não vai precisar, por exemplo, utilizar o live server. Primeiro eu vou instalar ele. Então npm install webpack dev server e vou instalar ele como uma dependência de desenvolvimento. Agora eu vou configurar ele. Depois de ter instalado ele aqui, eu vou agora no webpack.config.js e depois aqui de mode eu vou adicionar a configuração do dev server. É importante que você escreva exatamente igual porque são propriedades de configurações aqui. Então às vezes se por algum motivo não executar, dá uma olhada se está escrito exatamente igual. Eu vou dizer agora qual é o arquivo de entrada que ele tem que utilizar o nosso servidor. Eu vou então fazer isso através dessa propriedade que recebe um objeto e aqui eu defino qual é o diretório. Eu vou usar o path que já está importado aqui pra poder resolver pra mim esses endereços. E aí só pra reforçar, o path é interessante usar ele porque ele vai resolver os endereços independente do esquema que o sistema operacional usa. Se é barra invertida, se é barra normal, eu digo pra navegar pelas pastas. Muito bem, porque às vezes no Windows, MacOS e Linux é diferente. Então aí a gente tem também essa tranquilidade dele resolver isso pra gente de acordo com o ambiente. Vou usar o join e aí vou usar a dirname partindo da minha pasta que é a raiz. Maravilha, então definimos aqui o ponto de entrada. E posso também definir aqui em que porta eu quero rodar o servidor. Por exemplo, porta 3000. Você pode definir aqui a porta. Vou salvar e agora vou executar. npm run Ah, vamos criar um script pra executar aqui. Eu vou criar aqui dentro do paquete .json um novo script. Vou chamar de dev, vou adicionar aqui webpack serve. Então esse é o script pra executar o servidor do webpack que a gente acabou de instalar e configurar. Então agora a gente pode executar.
