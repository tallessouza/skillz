WEBVTT Nessa aula eu vou explicar pra você o que é o React, pra gente então começar a explorar essa ferramenta na prática, tá? O React é uma biblioteca JavaScript, ou seja, vamos utilizar como linguagem de programação o JavaScript pra construir interfaces. E o legal do React, uma curiosidade caso você não saiba, é que ele foi criado e é mantido pelo Facebook. Então se você usa o Facebook e tantas outras aplicações, você estava usando ali aplicações feitas com React e talvez não sabia. Bom, com o React a gente consegue criar interfaces interativas e dinâmicas com uma abordagem declarativa. Então a gente consegue declarar aquilo que a gente quer que a nossa aplicação tenha na tela. E você vai ver que inclusive tem muita coisa em comum com a forma de escrever HTML, que inclusive vamos também utilizar HTML, CSS e tudo mais dentro do React. O que o React vai fazer para a gente é facilitar o nosso trabalho para construir as nossas interfaces de forma interativa, dinâmica, utilizando essa abordagem declarativa. Outra grande vantagem de utilizar o React é que ele facilita a construção da nossa aplicação através de componentes que são reutilizáveis e que refletem mudanças de estado. O que isso significa? A gente consegue criar no nosso projeto componentes e esses componentes são partes que a gente consegue criar separadas para reaproveitar elas. A vantagem de criar componentes é que facilita também a manutenção, porque imagina só, digamos que você tenha um botão ali, um tipo de botão, que aparece aquele mesmo padrão, formato de botão em vários lugares. Ao invés de você criar esse botão 20 vezes, você pode criar ele uma vez como um componente e reutilizar ele em quaisquer lugares que você queira utilizar na sua aplicação. E facilita a manutenção porque se você precisa mudar alguma coisa nesse botão, se ele é um componente, você muda no componente e ele reflete essa alteração, essa mudança em todos os lugares que você usou aquele componente. Então trabalhar com componente dá muita produtividade para desenvolver, facilita a manutenção, o teu código fica mais organizado porque você separa até as responsabilidades em que cada componente tem ali um objetivo, cada componente tem a sua responsabilidade. E isso facilita muito também para a gente desenvolver aplicações escaláveis, uma vez que os componentes a gente consegue reutilizar eles. E o legal dos componentes também é que, além dessa estratégia de componentização, o React também trabalha com estados, então sempre que um estado muda, o estado tem um poder de refletir essa mudança ali na nossa interface no exato momento que esse estado muda e isso a gente vai ver na prática. E aqui, só para refrescar a sua memória, só para você relembrar, é Document Object Model, ou modelo de documento por objetos. É um modelo de representação de um documento HTML nessa estrutura que tem um formato de árvore. Então a gente tem ali o documento, dentro do documento HTML, dentro do HTML a gente tem a parte do head, dentro do head tem o título, por exemplo, e dentro do próprio título a gente tem um texto, dentro do body tem um H1, pode ter um A, um HF, né? Pode ter ali dentro do A o HF, e por aí vai um span, e assim por diante. Então, quando a gente inspeciona uma página na web, na verdade a gente está olhando para a DOM dessa página. E por que eu estou trazendo a DOM aqui? Porque como que o React reflete essa alteração em tempo real? Utilizando estados. O React ele trabalha com uma virtual DOM. A virtual DOM é uma representação em memória, ou seja, está carregado ali na memória da DOM real, permitindo que o React calcule de maneira eficiente as mudanças que foram feitas ali na interface do usuário. Então, quando o estado de um componente muda, o React cria um novo virtual DOM em memória para ele conseguir comparar, comparar com a DOM real. E assim ele consegue saber exatamente qual elemento mudou, qual componente mudou. Então ele não vai renderizar tudo de novo, mas ele vai renderizar somente aquilo que precisa. Olha só aqui essa ilustração que eu trouxe aqui para você do lado. Aqui a gente tem a Virtual DOM que está ali na do lado. Aqui a gente tem a virtual DOM, que está ali na memória, e aqui a gente tem a real DOM. Então, ele faz uma cópia da real DOM, ou seja, da DOM real, na memória, e ali, digamos que a gente adicionou, por exemplo, aqui, digamos que a gente adicionou um item novo na lista, e esse daqui já estava na lista, porém, ele foi modificado. Então, ele não foi um novo item, ele só foi modificado. O que o React vai fazer é comparar com a cópia que ele tem ali na memória e da modificação que foi feita em memória, ele vai comparar com a real DOM aqui, ou com a DOM real, melhor dizendo, para ele conseguir alterar só aquilo que precisa. E por isso a gente tem uma maneira muito otimizada de refletir as alterações na tela do usuário. Então, essa é a base do funcionamento do React e agora a gente vai utilizar ele na prática.
