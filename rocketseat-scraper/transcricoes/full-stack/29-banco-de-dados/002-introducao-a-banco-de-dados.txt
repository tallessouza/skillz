WEBVTT Nessa aula, vamos fazer uma introdução a banco de dados para você compreender os princípios e conceitos importantes de um banco de dados. Começando pela diferença entre dado e informação. São dois conceitos distintos, mas um depende do outro para acontecer. Os dados são materiais bruto. Um dado separado, isolado, não tem significado ou sentido. Por exemplo, 2006 é um dado numérico. Sozinho 2006 não significa muita coisa. 2006 é o quê? É um ano? É um número? É um código? Não sabemos, porque sozinho ele não tem sentido ou significado. A informação, por sua vez, já é uma percepção concreta e a gente extrai a informação dos dados. Temos que organizar os dados para extrair informações e a informação, por sua vez, entrega um sentido, uma percepção mais concreta. Aqui embaixo tem um exemplo. Veja que temos João. João é um dado de texto, que pode ser, por exemplo, o nome de um cliente. E 2016 é um outro dado numérico que pode ser o ano de nascimento do João, que é o cliente que a gente está usando como exemplo. Então veja que, organizando esses dados, nós temos que João nasceu em 2006. Conseguimos, inclusive, processar esses dados para extrair informação. Por exemplo, processando esses dados, conseguimos extrair a informação de que, atualmente, João possui 18 anos. Então perceba, organizamos esses dados, passou a fazer sentido e agora extraímos informação desses dados. Então essa é a diferença de dado e de informação. E qual é a finalidade, então, de um banco de dados? Um banco de dados, como você já pressupõe, armazena dados. A grande proposta e o grande objetivo de um banco de dados é uma coleção organizada de dados. Ou seja, banco de dados tem uma estrutura para organizar os dados. Muito bem, por que, então, utilizar um banco de dados? Já sabemos que o banco de dados organiza os dados e isso facilita a gestão, o acesso a esses dados, manipular esses dados para obter informação, na escalabilidade porque banco de dados também suporta grande volume de dados, na eficiência porque contribui também com a melhora na performance da aplicação porque uma vez que os dados estão organizados e o banco de dados tem um próprio mecanismo que a gente vai ver ali de leitura desses dados, de fazer pesquisa e tudo mais, consegue entregar essas informações, esses dados muito mais rápido para a nossa aplicação. E essa eficiência também está conectada com essa organização e graças a essa organização o acesso rápido a esses dados estruturados e acessíveis e integridade e confiabilidade porque os dados são consistentes e também protegidos. Os dados são consistentes porque nós definimos, você vai ver logo mais quando a gente for falar de banco de dados relacional, por exemplo, que a gente tem, cada coluna tem um tipo de dado. Então a gente define que a coluna vai armazenar número, que a coluna vai armazenar texto e aquela coluna vai armazenar apenas os valores do tipo que você determinou, isso para garantir consistência e o próprio banco de dados vai garantir que essas regras sejam atendidas e cumpridas ali. Então se um tipo de dado foi definido para guardar número, ele não vai aceitar texto e assim por diante. Então tudo isso para garantir consistência e manter ali seus dados protegidos, mantidos dentro do banco de dados. E agora que você já conhece o que é um banco de dados, vamos continuar avançando.
