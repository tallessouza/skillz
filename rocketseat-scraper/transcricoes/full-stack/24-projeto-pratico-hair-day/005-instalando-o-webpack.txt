Então, vou fazer aqui a instalação com o npm e, abreviação de install, eu vou instalar o Webpack com a versão 5.89.0, então eu recomendo que você use a mesma versão que eu. Então, eu coloco aqui o arroba na frente do Webpack e a versão e o Webpack CLI arroba 5.1.4, que é a versão que eu vou usar, e a flag traço-traço-save-traço-deve pra instalar como dependência de desenvolvimento. Vou dar um enter, terminou, se a gente olhar no paquete.json, vai estar aqui agora uma nova propriedade com as dependências de desenvolvimento que a gente acabou de incluir no nosso projeto. Muito bem, aproveitando que a gente está aqui no arquivo paquete.json, eu vou adicionar um script novo, vírgula aqui então no final do script de server, eu vou adicionar aqui um script de build, pra poder gerar build da nossa aplicação com o Webpack, e aqui eu vou só acrescentar o próprio Webpack, então quando eu executar build, eu quero que execute o Webpack. Dentro da nossa pasta src, eu vou criar um arquivo novo, deixa eu só fechar o paquete.json, porque a gente não vai precisar dele por enquanto. Então, dentro da src eu vou criar um novo arquivo que vai ser o main.js, esse vai ser o nosso arquivo de entrada da aplicação, e todo o restante da nossa aplicação vai partir daqui, então é uma forma da gente centralizar o ponto de entrada, também muito conhecido como entry point, e aí a partir daqui segue o restante da nossa aplicação. Por enquanto, eu vou deixar esse arquivo vazio mesmo, agora a gente pode criar um novo arquivo na raiz do nosso projeto, fora da pasta src, eu vou chamar de webpack.config.js, e aqui a gente vai definir as configurações do Webpack. Eu vou importar o path igual a recard, e o path aqui. Aqui a gente tem que usar essa maneira de importar, utilizando essa estratégia, não aquela síntese do import e export, tem que usar essa síntese, porque a gente vai rodar usando o próprio Node, então quando a gente está rodando JavaScript fora do navegador, instalando pacotes, usando aqui o próprio npm para gerenciar as dependências, a gente está usando aqui o JavaScript no ambiente Node, então ele vai rodar do lado do servidor digamos assim, e muita coisa vai depois pronta para o navegador, quando a gente está usando por exemplo bibliotecas e tudo mais, então ele roda aqui na nossa máquina, no ambiente usando o Node, e aí por isso a gente tem que usar essa síntese nesses arquivos de configuração, como é o caso aqui do Webpack.config.js. Então agora eu vou pegar o módulo.export, que a gente vai exportar essas configurações. Vamos começar definindo aqui o nosso target, que é o web, quando a gente está desenvolvendo para web, e aqui no Node eu vou colocar que o modo que a gente está usando agora é desenvolvimento. Eu vou separar aqui para ficar mais fácil de visualizar. A gente vai definir agora onde está o nosso arquivo de entrada da nossa aplicação, por isso que eu criei esse main.js, eu vou usar o path.resolve para resolver então esse caminho, vou usar aqui o dirname, porque ele parte da própria raiz aqui do projeto, vou acessar a pasta src e dentro dessa src é o main.js, que a gente vai utilizar. É mais recomendado utilizar o path, porque ele já vai resolver a navegação de acordo com o sistema operacional, Linux, macOS, Windows, às vezes muda ligeiramente a forma de navegar entre esse sistema operacional. Se você coloca aqui a navegação mais manual, por exemplo, ponto, barra, dois pontos, e aí vai, tipo o nome da pasta, como a gente fez ali, e aí se você mudou o ambiente, o sistema operacional, pode ter dificuldades ali de entender essa importação. Então, por isso que é mais recomendado usar o path. E agora, o output, que é a saída, quando a gente for fazer a build. Para o filename, que é o nome do arquivo, eu vou usar aqui o main.js também, e para o path, eu vou dizer aqui qual é o diretório que eu quero ter de saída. Eu vou usar também aqui um path.resolve, vou usar o dirname e aí eu vou pedir para ele gerar a pasta dist, então ele vai colocar dentro dessa pasta dist aí o resultado da nossa compilação e vou salvar. Vamos testar? Eu vou abrir o terminal, deixa eu limpar aqui, eu vou executar o comando npm run build, que é o script que a gente criou para gerar o webpack, e aí ele vai, o webpack vai gerar a build da nossa aplicação usando essas configurações que a gente definiu aqui no webpack.config. Eu vou fechar aqui, vou dar um enter, e terminou. Pode ver, ele não apareceu aqui, mas se eu clicar aqui nesse botão de fazer um reload, ele aparece.
