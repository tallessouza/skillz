WEBVTT Nessa aula você vai compreender o que é um ORM. Vamos começar pela definição de um ORM que vem de Object Relational Mappers, traduzindo mapeador objeto relacional e você já vai entender o porquê desse nome. Bom, um ORM é uma ferramenta que facilita o trabalho com banco de dados e ele permite interagir com o nosso banco de dados sem escrever consultas SQL diretamente, ou seja, a gente vai manipular os dados dentro do nosso banco de dados através de objetos. Daí vem esse nome de mapeador objeto relacional e o relacional porque a gente utiliza bancos de dados relacionais, é daí que vem o nome. Com o ORM a gente manipula as tabelas e os dados do banco como se fossem objetos, ou seja, através de objetos dentro da linguagem de programação que a gente está utilizando. Isso simplifica a leitura, escrita e também a manutenção do nosso código e ele também cuida da conversão dos dados entre o nosso banco de dados relacional, ali as nossas tabelas e o objeto que a gente está utilizando para interagir com o nosso banco de dados. Então daí vem esse nome de mapeador objeto relacional ou simplesmente ORM como é mais comumente utilizado e chamado. Aqui tem uma representação dessa interação. Veja que no nosso projeto a gente pode representar o usuário com um objeto, tendo ali um ID, um nome e um e-mail e a gente tem do lado do banco de dados o usuário sendo representado ali dentro de uma tabela, então a gente tem a tabela de usuários e dentro da tabela de usuários a gente tem vários usuários. Aqui eu coloquei apenas um como exemplo, então tem ali o mesmo usuário ID12, nome Rodrigo e esse e-mail aqui de exemplo e a gente consegue dentro aqui do lado do nosso projeto, do lado da nossa aplicação manipular esse usuário aqui que está dentro de uma tabela de usuários como um objeto sem escrever código SQL, a gente manipula como um objeto dentro ali da nossa linguagem de programação e o ORM é quem faz esse meio de campo de converter esse objeto lá dentro do banco de dados para código SQL e vice-versa dentro da nossa aplicação esse código SQL a gente usa, não escreve o SQL diretamente, mas a gente manipula com objetos, então a gente tem um RMA no meio do campo fazendo então essa conexão entre os objetos que a gente está utilizando ali para manipular os dados e o banco de dados, os dados ali, as tabelas dentro do nosso banco de dados. Quais são as vantagens de utilizar um ORM? Bom, menos código SQL manual, a gente não escreve código SQL, facilita muito a manutenção e é compatível com múltiplos bancos de dados, digamos que você esteja trabalhando com SQLite e depois você precisa mudar de banco de dados para um banco de dados púltipos, por exemplo, é basicamente você mudar na configuração o banco de dados que você vai utilizar, como você tem a parte ali de conexão com o banco de dados e a manipulação do banco de dados por conta do RM, então se você configura para usar um outro banco de dados, você não precisa reescrever o teu código porque você usou o RM e o RM vai cuidar então de tudo pra você, então essas são vantagens de utilizar o ORM. E qual é a diferença de um RM para um care builder? São diferentes, a diferença principal entre um RM e um care builder é o nível de abstração, como eles interagem com o banco de dados, o RM converte tabelas do banco de dados em objetos dentro da linguagem de programação e você usa os objetos e o RM faz o SQL pra você, com um care builder você consegue ter ajuda do próprio care builder para construir as consultas SQL, usando o seu próprio código, a linguagem de programação sem escrever também SQL diretamente, mas também é possível escrever SQL diretamente dentro de um care builder, por exemplo, e aí no care builder você tem mais controle, mas também você tem um pouquinho mais de trabalho para construir a lógica das suas consultas, então basicamente a diferença entre eles é o nível de abstração, então o RM ele tem um nível de abstração maior, ele vai cuidar mais pra você ali do banco de dados, dessa interação com o banco de dados, deixando você manipular o banco através de objetos, mas no final das contas, ambos atendem o mesmo propósito, que é você conseguir manipular dados dentro do seu banco.
