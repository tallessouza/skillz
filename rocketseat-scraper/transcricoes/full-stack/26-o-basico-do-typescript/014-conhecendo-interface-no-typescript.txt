WEBVTT Nessa aula, você vai aprender a utilizar interfaces. Vamos criar uma função chamada newProduct e digamos que essa função vai cadastrar um novo produto. E para cadastrar um novo produto, aqui eu preciso receber os dados desse produto. Só que esse produto não vai ser só uma string, porque produto pode ter um código, pode ter um nome, um valor e assim por diante. Então, o produto tem muitas informações e aqui a gente pode usar algumas delas como exemplo para criar um tipo customizado, uma tipagem customizada para atender esse produto. Ou seja, quais são as informações que esse produto precisa. E para poder criar uma tipagem personalizada para produto, a gente pode utilizar interfaces. Para criar uma interface, eu começo aqui declarando que eu estou criando uma interface e aqui eu vou dar um nome para a interface. É uma boa prática escrever nomes de interfaces com a primeira letra maiúscula. Por exemplo, PRODUCT. Você vai ver que até a cor fica diferente. Outra prática muito comum de ser encontrada nas aplicações é colocar uma letra I só para identificar que é uma interface, mas aí vai também de gosto e de costume. Aqui eu não vou colocar essa letra I não, eu vou criar a nossa interface como a primeira letra maiúscula seguindo essa boa prática e ela vai chamar PRODUCT. Aqui a gente abre e fecha as chaves. E aqui dentro a gente tem o corpo da nossa interface e é aqui dentro que a gente vai definir quais são as propriedades que um produto deve ter. Por exemplo, digamos que o produto tenha um ID, eu vou falar aqui que o tipo é NUMBER. Ou seja, o ID vai receber valores numéricos. Coloco vírgula para adicionar mais uma propriedade e eu quero adicionar, por exemplo, o nome do produto e digamos que o nome é uma string. Então pronto, tenho o ID e tenho aqui o nome. E aí, colocando vírgula, você poderia adicionar mais propriedades. Como eu vou adicionar só essas duas aqui de exemplo, posso deixar a última aqui sem vírgula, sem nenhum problema. Agora, ao invés de o produto ser uma string simples, eu posso agora substituir a tipagem por PRODUCT e utilizar a própria interface que eu acabei de criar. Então veja que agora o nosso produto aqui tem uma tipagem customizada, criada especialmente aqui para atender as especificidades, as características desse produto. Então você pode utilizar as interfaces para criar os seus próprios tipos para atender as necessidades dos dados que a sua variável precisa ter, por exemplo. E aqui agora a gente tem, dentro do console.log, a gente pode utilizar aqui, por exemplo, PRODUCT. Não precisa ser dentro do console.log, tá? É só mesmo de exemplo. PRODUCT. Tá vendo?
