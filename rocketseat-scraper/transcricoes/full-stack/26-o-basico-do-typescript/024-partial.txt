WEBVTT Nessa aula você vai aprender a utilizar o Partial do TypeScript. Vamos criar aqui uma interface User para representar os dados de um usuário. Vou colocar um ID do tipo Number, o Name como String e e-mail também como String. Digamos que nós queremos criar aqui um novo usuário, então vou utilizar aqui New User e aqui vou colocar dois pontos para poder utilizar a tipagem User. Igual e aqui a gente vai então colocar os dados desse novo usuário. Como a gente está usando aqui a tipagem User, a gente precisa então, olha só, atender os requisitos, satisfazer essa tipagem. E se eu passo o mouse aqui, ele logo me mostra que eu preciso informar ID, nome e e-mail. Então eu posso passar aqui para o ID como um número sendo 1, para o Name posso passar o meu nome e veja que enquanto eu não preencher tudo, ele vai ficar reclamando. Quem que está faltando aqui? Está faltando o e-mail, certo? Se eu passo o mouse aqui, ele me mostra que eu esqueci de informar o e-mail, a não ser que o e-mail seja opcional. Quando eu quero fazer uma propriedade ser opcional, a gente pode usar antes dos dois pontos, Interrogação e veja que ele não vai reclamar mais, porque o e-mail é opcional. Eu posso tanto informar ele, veja só que ele aparece aqui, uma interrogação, ou posso simplesmente não informar, porque é opcional. Só que aqui eu vou deixar todos os campos obrigatórios, incluindo o e-mail. Então vamos colocar aqui, e-mail, e aqui eu vou colocar como rodrigo, arroba, e-mail.com de exemplo. Agora a gente não tem mais o New User reclamando, porque todas as propriedades de um usuário foram informadas, ou seja, a tipagem foi atendida. Agora digamos que eu queira atualizar um usuário, então eu quero atualizar o Updated User, por exemplo, e eu vou definir aqui também que eu quero utilizar o tipo de New User. Só que o que acontece agora? Para atualizar um usuário, digamos que eu não quero ter que informar todos os campos de novo. Digamos aqui que eu queira mudar só o nome, então eu vou colocar aqui, name, dois pontos, e eu cadastrei só com o Rodrigo, mas digamos que eu queira atualizar incluindo o meu sobrenome, Rodrigo Gonçalves. Mas perceba que aqui, se eu passo o mouse, ele diz que eu esqueci de informar o ID e o e-mail. Por quê? Como eu estou utilizando aqui o User, e todas as propriedades do User é obrigatória, automaticamente eu vou ter que informar todas as propriedades aqui também, para satisfazer essa tipagem, igual a gente fez aqui em cima. É aí que entra então o Partial. Podemos utilizar aqui o Partial para deixar todas as propriedades aqui dentro opcionais. Então, ao utilizar o Partial, todas as propriedades aqui vão ficar opcionais, e com isso eu posso atualizar aqui a propriedade que eu quiser, ou seja, definir aqui dentro a propriedade que eu quiser atualizar, e não ter que preencher todas de novo. Então, para utilizar o Partial, a gente pode utilizar aqui o sinal de menor e maior, para envolver aqui o User, e aí a gente coloca aqui Partial. Agora perceba que ele não está reclamando mais, e aí se eu passo o mouse aqui, ou posso até colocar aqui a vírgula contra o barra de espaço, está vendo que agora o e-mail tem a interrogação, e o ID também a mesma coisa, ele tem uma interrogação. Veja que agora todas as propriedades de User são opcionais. Então, cabe muito bem nesse contexto em que eu não preciso informar tudo de novo. Eu posso informar aqui somente as propriedades que eu quero atualizar, nesse caso é só o Name. A interface User aqui foi preservada, perceba que aqui todos os campos são obrigatórios, nada mudou aqui, inclusive aqui se eu tiro, por exemplo, o ID, veja que aqui ele reclama. O que muda aqui no nosso UPDATE USER é que aqui a gente está utilizando o Partial, para dizer que a gente quer sim utilizar o tipo User, mas utilizar o tipo User com todas as propriedades opcionais. Então, esse é o utilitário Partial que você pode utilizar nas suas aplicações quando você tem esses cenários, que você quer deixar as propriedades de um tipo opcionais. E a vantagem disso é que eu não precisei criar uma nova tipagem, duplicar essa tipagem ou deixar todos os campos opcionais, mas com o Partial eu consegui reaproveitar essa tipagem para atender esse cenário aqui.
