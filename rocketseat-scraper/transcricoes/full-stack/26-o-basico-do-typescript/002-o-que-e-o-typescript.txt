WEBVTT Nesta aula eu vou explicar para você o que é o TypeScript, mas antes vamos falar de uma característica importante do JavaScript. O JavaScript é uma linguagem não tipada, o que isso significa? Significa que a gente não precisa definir um tipo quando a gente cria uma variável, por exemplo. Existem linguagens de programação que têm essa característica de ser tipadas, ou seja, se você cria uma variável, você precisa declarar, definir qual é o conteúdo que aquela variável vai armazenar e ela só vai aceitar aquele conteúdo que você definir. O JavaScript, por sua vez, é uma linguagem não tipada, então a gente não precisa criar uma definição, a gente não cria uma variável e diz para a variável que ela vai guardar só números ou que ela vai armazenar somente texto. No JavaScript, uma variável pode receber conteúdo de tipos diferentes. Por exemplo, a gente pode criar uma função que espera números como parâmetros, mas podemos passar uma string que é texto ou vice-versa. Você pode criar uma variável e atribuir para ela, para a mesma variável, um número ou, por exemplo, um texto. Então, a variável no JavaScript, por exemplo, e assim como os parâmetros, ela tem essa característica de flexibilidade. Então, no momento que a variável recebe o valor, ela vai simplesmente guardar aquele valor. Essa característica bastante flexível do JavaScript é o que inclusive tornou o JavaScript bastante popular, mas pensando no crescimento consistente da aplicação, é importante definir regras e é aqui que entra o TypeScript. O TypeScript adiciona tipagem no JavaScript. No final, o código TypeScript é convertido em JavaScript, permitindo ser executado em qualquer lugar que o JavaScript é executado. Ou seja, o TypeScript é um recurso que a gente utiliza enquanto está desenvolvendo a nossa aplicação. Mas, no momento que a gente vai fazer a build da nossa aplicação, vai publicar ela, vai disponibilizar a nossa aplicação, todo o código vai ser convertido para JavaScript. Tá certo? Então, o TypeScript é uma ferramenta que a gente utiliza enquanto está desenvolvendo a nossa aplicação, porque o TypeScript é útil para nós, enquanto estamos desenvolvendo, para ajudar a gente a identificar problemas de tipagem, parâmetros, já antecipar alguns erros que a gente poderia descobrir ali no momento de execução da nossa aplicação, enquanto está executando a nossa aplicação e assim por diante. Bom, o TypeScript se torna JavaScript, como eu comentei, e basicamente passando por essas três etapas. Então só para reforçar que o TypeScript é utilizado enquanto a gente desenvolve a nossa aplicação. E quais são as vantagens, então, de utilizar o TypeScript?
