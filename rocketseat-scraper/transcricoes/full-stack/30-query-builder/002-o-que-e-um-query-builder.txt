WEBVTT Nessa aula vou explicar pra você o que é um Care Builder, pra você compreender o conceito, pra gente utilizar na prática, tá? Bom, um Care Builder permite construir instruções SQL, ou seja, o código SQL, independente do banco de dados. Existem diversos bancos de dados relacionais, por exemplo, MySQL, Postgres, e cada banco, embora os bancos de dados relacionais utilizam o SQL como linguagem padrão, mas pode ter alguns detalhes ali no uso do SQL em cada contexto de banco de dados. Por exemplo, no Postgres pode ter um detalhezinho ali da maneira como se utiliza o SQL, talvez um ponto e vírgula no final que é obrigatório, enfim, alguma palavra reservada diferente, mas em essência é SQL, isso é importante de ser dito, mas pode ter alguns detalhes ali diferentes na hora de utilizar o SQL dentro de cada banco de dados. Bom, é aqui que entra então o Care Builder, porque ele abstrai pra gente, ou seja, você não precisa se preocupar com o banco em que o SQL vai ser executado, porque essa responsabilidade fica com o Care Builder, a gente utiliza código SQL através de métodos, ou seja, a gente não escreve o SQL direto, a gente utiliza métodos do Care Builder, por exemplo, pra método de selecionar, método de inserir e assim por diante, e o Care Builder conhecendo qual é o destino, o banco de dados que esse SQL vai ser executado, ele então gera o código SQL que deve ser executado dentro do banco de dados que a gente está utilizando, sejam Postgres, MySQL, SQLite e assim por diante, pra conseguir aquele resultado que a gente está esperando, executar aquilo que a gente quer. Então, em resumo, a gente tem esse esquema aqui, temos o nosso Care Builder, ele é responsável então por fazer essa abstração, ou seja, eu não preciso saber qual o código SQL, ou eu não preciso necessariamente tomar conhecimento do código SQL que vai ser gerado, eu vou usar métodos do Care Builder, para selecionar, deletar, atualizar e assim por diante, baseado nas configurações do Care Builder, ele sabe qual é o banco de dados que eu estou trabalhando, e ele vai gerar então o SQL de acordo com esse banco de dados, para então executar esse código SQL dentro do banco de dados que a gente está trabalhando. Então, veja que aqui a gente tem o Care Builder cuidando pra gente dessa abstração, e o Care Builder ele é interessante porque a legibilidade também é bem interessante, os métodos de inserir e de selecionar, a gente vai adicionando e utilizando métodos e a legibilidade, a forma de ler ali o que a gente está fazendo com o Care Builder também é bem interessante e é uma legibilidade muito boa. Bom, agora que você já conhece o conceito de um Care Builder, a gente vai começar a utilizar ele na prática.
