WEBVTT Nessa aula eu vou explicar pra você o projeto que a gente vai desenvolver juntos a partir de agora. E o projeto que a gente vai desenvolver nessa etapa é uma API para pedidos para um restaurante. Até coloquei aqui uma ilustração, então digamos que é pra esse restaurante aqui que a gente vai desenvolver essa API, tá? Agora deixa eu explicar pra você o fluxo da nossa aplicação. Bom, quando o cliente chega no restaurante, então nós vamos abrir uma mesa, tá, pro cliente fazer pedidos. Então o cliente vai fazendo pedidos ali relacionados então a uma mesa. A gente precisa controlar a abertura e o fechamento dessa mesa, ou seja, enquanto a mesa está aberta significa que ela está ocupada, ou seja, tem clientes ali naquela mesa fazendo pedidos, ou seja, consumindo. Quando o cliente encerra a conta, ou seja, o cliente terminou a consumação, ele vai encerrar a conta pra ir embora, o restaurante vai fechar a mesa e nesse fechamento ali, por exemplo, obter o valor total da conta pra fazer o pagamento e etc, tá? E quando a mesa é fechada, então a mesa fica disponível novamente. E aí volta aqui pro início, porque agora a mesa está disponível novamente pra que outro cliente faça consumo nessa mesa. Então veja que a gente tem aqui um ciclo e ao longo do dia muitos clientes podem passar pela mesma mesa ali pra fazer o consumo, beleza? Então esse é o fluxo da nossa aplicação e a gente vai então utilizar as seguintes tabelas. Deixa eu mostrar pra você agora as tabelas que nós vamos utilizar no nosso banco de dados. A gente vai ter uma tabela para guardar as informações dos dados do produto, como o nome do produto, o preço do produto, e além da tabela pra guardar os dados do produto, também teremos uma tabela pra guardar os dados das mesas, que eu vou chamar de tables aqui, essa tabela. Então o restaurante pode ter muitas mesas, a gente vai ter ali, por exemplo, o número da mesa pra identificar cada mesa. Bom, além das tabelas de produtos e de mesas, também teremos uma tabela pra guardar os dados dos pedidos. Então imagina só que o cliente pediu ali uma bebida, um prato, uma porção, então a gente precisa ter uma tabela para guardar os pedidos. E teremos por fim uma outra tabela, session tables, pra guardar essas sessões da mesa que abre e encerra pra que a gente vincule os pedidos a um uso ali da mesa ali durante esse consumo. Então nessa abertura e fechamento. Então perceba que aqui a gente vai conectar essas tabelas pra que a gente consiga então fazer toda a nossa aplicação bem integrada, bem consistente aí, inclusive utilizando relacionamentos, enfim, vai ser bem legal e a gente vai começar a desenvolver então essa aplicação na prática, utilizando tudo que você aprendeu até aqui, tanto de relacionamentos, banco de dados, utilizando o Express pra criar APIs e etc. Então vai ser bem massa, então bora codar!
