WEBVTT Existem dois comandos super essenciais para você trabalhar com repositórios remotos. É o comando git pull, ele vai puxar modificações do seu repositório remoto aqui para o seu repositório local. Um exemplo, se você está trabalhando com um time, as pessoas estão trabalhando naquele projeto, elas vão enviar as modificações lá para o repositório remoto e você ter que trazer para cá para o seu projeto, para que o que você estiver fazendo aqui também não haja conflito com as coisas que já fizeram lá, ok? Existem até maneiras do git gerenciar conflitos, caso você trabalhe na mesma linha de código que uma outra pessoa trabalhou. Então ele tem uma maneira de trabalhar com isso, é só para você entender que é seguro que você primeiro puxe as modificações que tem no repositório remoto. Aí você fez as suas modificações aqui, você precisa enviá-las para o seu repositório remoto, do seu repositório local lá para o remoto. Você vai usar o comando git push, a gente vai entender melhor isso na prática.
