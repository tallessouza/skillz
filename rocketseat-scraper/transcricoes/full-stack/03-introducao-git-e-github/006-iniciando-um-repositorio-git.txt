WEBVTT Nessa aula nós vamos iniciar um git em algum projeto, ok? Vamos supor que você tenha uma pasta em algum lugar no seu computador. Nesse caso eu tenho uma pasta aqui chamada git-aula. Dentro dela eu tenho apenas um arquivo que eu chamei de frases.txt. Nem importa muito o arquivo ou o conteúdo do arquivo, ok? São apenas algumas frases aqui. Agora o que eu vou precisar? Abrir o meu terminal. Vai depender da máquina que você está utilizando. E eu preciso navegar agora, eu preciso ir até esse local, até essa pasta que eu estou trabalhando aqui com o git. Que eu vou iniciar o git. Existem algumas maneiras da gente fazer isso. Quer um exemplo? Aqui no Mac a gente usa o cd. O cd você vai usar também no git bash no Windows e vai usar também no Linux, ok? Cd significa change directory ou mude de diretório. Então significa que eu vou pedir para mudar de diretório. Agora eu preciso saber aonde é que está essa pasta no meu computador. Aqui no Mac é até que simples, porque eu posso arrastar a pasta para cá. Infelizmente não é assim que funciona em todos os sistemas operacionais. Por isso você precisa ter uma clareza de onde você está colocando a sua pasta para você trazê-la para cá. Quer um exemplo? Se você abrir o Windows ali o c:// e criar uma pastinha ali chamada git-aula, provavelmente a sua vai estar no c://git-aula. O Windows também eu preciso lembrar, mas o Windows pode ser que ele faça dessa forma, né? Git-aula. Só que pode ser que ele não esteja ali. Já em sistemas Unix a gente tem como navegar a partir da raiz do computador, que é a raiz do HD, que é uma barra. E aqui eu colocaria o nome dos usuários, eu procuro o meu usuário, procuro aqui o desktop, que é onde eu estou, e git-aula, que é aonde eu quero ir. Tudo bem? Então esse processo de terminal é um processo fundamental para que você possa fazer essa pequena navegação. Ok? Então é importante que você saiba como chegar na pasta. Pronto, consegui chegar na pasta. O que você vai fazer? CD, vai colocar o nome e vai dar o Enter. E nem importa qual terminal você está usando, tá pessoal? Se o seu está colorido ou não, se o seu tem outras informações, não é isso o importante. O importante é você chegar na pasta. Chegou na pasta, você pode conferir se está nela, colocando PWD. PWD vai mostrar o caminho até essa pasta. E o caminho é, nesse meu caso, da raiz do computador, que é essa barra aqui, usuários, Mike Britto, desktop, git-aula. Então agora eu estou no meu projeto. E o fundamento dessa aula é como que eu inicio um repositório git no meu projeto. É o jeito mais simples do mundo. Git, init. Só isso. Dá o Enter. Acabou. Você já tem um repositório iniciado aqui. Literalmente, você não viu nada de diferente, parece que não aconteceu nada, mas agora você já tem um repositório git nesse projeto.
