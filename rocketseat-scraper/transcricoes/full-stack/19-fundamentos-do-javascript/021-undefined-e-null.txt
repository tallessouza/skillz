WEBVTT Nessa aula vamos ver na prática a diferença de Undefined e de Null, tá? Vamos começar pelo Undefined. O Undefined é o indefinido, tá? Vou até colocar aqui ó entre parênteses, indefinido, ou seja, é um tipo de dado que o JavaScript atribui de forma automática para uma variável que ainda não tem nenhum valor, por exemplo, eu vou criar uma variável aqui, eu vou chamar ela de Emptiness, ou seja, vazia. Se eu pedir aqui para exibir o console.log e colocar o valor é dois pontos, olha só, eu vou colocar vírgula aqui para o console colocar na frente e vou salvar, tá vendo? O valor é Undefined. A variável existe, a variável foi declarada, mas ela não tem nenhum valor, então o valor que é definido por padrão para uma variável é o Undefined, até que eu mude esse valor. Se eu colocar aqui, por exemplo, um texto com o meu nome, aí mudou, agora ela tem conteúdo. Como ela não tem ainda, ela é Undefined. Agora, o Null é um tipo de dado para dizer que o conteúdo de uma variável é vazio, ou seja, não tem nada dentro do objeto e o Null a gente coloca intencionalmente, que é quando você quer deixar claro que não tem nada ali dentro. Por exemplo, se eu colocar aqui ó, vou criar uma variável chamada Empathy e eu vou pôr Null nela, estou dizendo que essa variável está vazia, intencionalmente. E aí, se eu colocar aqui um console.log e eu vou colocar a mesma frase, tá? O valor é dois pontos, vou usar o Empathy e olha só, o que a gente vai ter aqui vai ser o Null, tá? Deixa eu aumentar um pouco mais o zoom para você visualizar melhor, olha só, tá vendo? Null é o que a gente tem. Então, essa é a diferença do Undefined para o Null.
