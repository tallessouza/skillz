Os tipos de dados, eles são importantes para definir qual é o conteúdo de uma variável e, para a linguagem, é importante saber se é número, saber se é texto e assim por diante. Até porque, quando a gente vai fazer uma conta, por exemplo, um cálculo matemático, a gente vai fazer com número e não com texto. Então, por isso que os tipos são importantes para a própria linguagem de programação e, para nós também, conseguir manipular da melhor maneira possível e fazer as operações que a gente quer, seja com número, com texto ou com outros tipos de dados que a gente vai ver a partir de agora. Então, vamos ver os tipos de dados no Javascript. O Javascript é uma linguagem dinâmica com tipos dinâmicos e é muito importante você saber disso porque isso significa que o tipo da variável é definida dinamicamente pelo tipo do valor atribuído. Você já deve ter percebido que, quando a gente vai criar uma variável no Javascript, a gente não diz qual é o tipo de conteúdo que a gente quer guardar na variável. Em outras linguagens de programação, isso não é possível. No momento que você declara a variável, tem linguagem de programação que você é obrigado a dizer qual é o tipo de conteúdo que você quer guardar naquela variável. Se você cria uma variável do tipo string, que vai guardar texto, ela só vai aceitar textos naquela variável. E o Javascript, por ser uma linguagem dinâmica com tipos dinâmicos, a gente pode, inclusive, modificar, reatribuir uma mesma variável com um tipo diferente. Você pode criar uma variável que guarda número agora, mas você pode trocar por um texto depois. Isso é possível porque ela é dinâmica. E o que define o tipo da variável é o próprio conteúdo. Por isso que a gente não cria uma variável e diz olha, essa variável é um texto. Quando a gente atribui um texto àquela variável, ele sabe que o conteúdo que está lá dentro é texto e, portanto, ela passou a ser uma variável do tipo texto. Como eu amo analogia, eu trouxe aqui mais uma e dessa vez de um copo. Pensa no copo como uma variável. Ele tem um único objetivo, o copo, que é guardar ali o seu conteúdo para não escorrer, para não vazar, desde que ele não esteja furado, mas enfim, ou quebrado. Mas pensa nesse objetivo do copo. Agora, o conteúdo é você quem vai definir. Você vai colocar água? Vai ser refrigerante? Não sei. O copo continua sendo o mesmo, mas você pode até colocar uma água agora e, talvez, colocar um refrigerante depois. Então, o objetivo do copo não muda, que é guardar o conteúdo. O que pode mudar, e você tem essa flexibilidade, é o que você vai guardar e armazenar ali dentro. Vamos começar a falar dos tipos agora, começando pelos tipos primitivos. Tipos primitivos são aqueles tipos que já estão disponíveis na própria linguagem e que você pode utilizar. E olha só que legal, a linguagem de programação, isso acontece com todas. Ela tem os seus tipos primitivos, que são esses que já estão disponíveis para você utilizar e você também pode criar os seus próprios tipos. A gente vai ver isso mais à frente, mas vamos focar agora nesses tipos que já estão disponíveis para a gente, que são os tipos primitivos. No Javascript, a gente tem String, esse, inclusive, que eu vou mencionar agora, são os principais valores primitivos do Javascript, que são aqueles que a gente vai usar com mais frequência. Então, começando aqui pela String, que é um tipo para se referir a texto. Então, uma variável do tipo String guarda texto. O Number, que é número. A gente tem o booleano, que é para valor verdadeiro ou falso, o famoso True ou False. A gente tem o Nulo. O Nulo é um tipo de dado para dizer que o conteúdo de uma variável é vazio ou que não tem nada dentro ali, seja dessa variável ou de um objeto, por exemplo, que a gente vai ver mais à frente. O Undefined é um tipo de dado que o próprio Javascript atribui automaticamente para uma variável que ainda não armazena valor nenhum. A gente viu, por exemplo, quando a gente estava estudando Hoisting e usar, por exemplo, a declaração de uma variável, usar uma variável antes dela ter sido declarada, por exemplo, que ele retornava que o conteúdo era Undefined. Exatamente por isso, porque o Hoisting levava a declaração para o topo, ali, e não tinha ainda um conteúdo atribuído à variável no momento que a gente acessava ela antes da declaração. Então, o Undefined é isso, que é atribuído automaticamente pelo próprio Javascript para uma variável que ainda não tem ou não armazena um valor.
