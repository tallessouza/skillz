Até coloquei essa imagem desse gancho de um guindaste mesmo, porque a ideia é essa, o JavaScript tem esse comportamento de pegar as declarações das variáveis e funções e levar para o topo do escopo onde elas foram definidas, e esse comportamento possibilita usar uma variável ou uma função antes mesmo que ela tenha sido definida. Você tem que definir a variável, tem que definir a função para poder usar ela, claro, mas você consegue usar ela dentro daquele escopo antes mesmo dela ter sido declarada. Isso é possível graças a esse comportamento de hoisting do JavaScript. Isso por exemplo não é possível num C Sharp da vida, ou em outra linguagem de programação que você tem que usar sempre uma variável ou uma função depois de ter declarado ela. Como o hoisting leva as declarações para o topo do escopo, no JavaScript você consegue usar uma variável ou função antes mesmo dela ter sido declarada. Lembrando que ele faz isso no momento que o JavaScript vai ser interpretado antes mesmo do código ser executado. Vamos dar uma olhada no hoisting de variáveis. Todas as declarações de variáveis são movidas para o topo do escopo, independentemente de onde tenha sido declarada dentro daquele escopo, ela vai estar disponível para ser utilizada ali no escopo onde ela foi definida. Importante, mesmo que as declarações das variáveis sejam movidas para o topo do escopo, é interessante usá-las depois delas terem sido declaradas, por quê? Caso contrário, se você usa uma variável antes dela ter sido declarada, você vai receber como conteúdo essa referência aqui de undefined, de referência indefinida. Isso é interessante porque com essa característica do JavaScript você consegue até organizar o seu código de uma forma mais intuitiva, definindo até a sequência ou a ordem das funções que você desejar.
