Mas antes da gente prosseguir aqui já criando nosso projeto, antes de eu falar um pouquinho o que vai ser esse projeto que a gente vai estar criando aqui hoje, basicamente o que é o Node? O Node é uma plataforma, então antes de mais nada, o Node não é uma linguagem de programação, ele utiliza o JavaScript por baixo dos panos, que é a linguagem de programação que a gente vai estar utilizando. Mas o Node é uma plataforma que permite a gente executar JavaScript pelo lado do servidor, ou seja, para fazer operações que outras tecnologias comumente já faziam, como por exemplo PHP, Java e por aí vai. O Node surgiu por volta de 2011, eu posso estar errado um pouquinho na data, mas 2010 talvez, não lembro exatamente quando que foi a data. Mas ele surgiu por um rapaz chamado Ryan Dahlke, ele pegou basicamente a ideia de pensar, se os browsers entendem JavaScript para conseguir utilizar o JavaScript para manipular nossa interface, colocar elementos em tela, remover elementos em tela, por que eu não consigo remover do browser talvez esse mecanismo que entende o JavaScript? e colocar numa plataforma para entender JavaScript para outros propósitos. E assim surgiu o Node. Hoje o Node é utilizado por baixo dos panos a V8, que é a engine do Chrome. Então, quando o Google Chrome surgiu, a grande revolução do Google Chrome, perante outros navegadores como o próprio Firefox ou até o Internet Explorer, era a V8, que era um motor de JavaScript, ou seja, que conseguia ler, entender JavaScript e interpretar JavaScript de uma maneira muito performática. O Node por baixo dos panos, então, conseguiu adaptar essa V8 que rodava dentro do Chrome para fazer a manipulação do JavaScript, para a interface, para a front-end, para então utilizar esse JavaScript para a construção de servidores HTTP, de aplicações para CLI, de aplicações mobile. Então, hoje, por mais que a gente veja o Node como uma tecnologia que é utilizada para a construção de aplicações HTTP, o Node é utilizado para uma variedade de coisas, ou seja, uma variedade de ferramentas que precisam executar JavaScript, mas que não estão dentro do ambiente do browser. Então, por exemplo, o React Native para construção de aplicativos móveis usa o Node, não só o React Native, né? A gente pode listar qualquer outra tecnologia de desenvolvimento mobile baseada em JavaScript, a gente vai pensar o próprio Ionic Framework, o React Native, o Native Script, tudo usa Node por baixo dos panos. Os grandes frameworks como o próprio Next.js lá do React, o próprio React, o Vue, Angular, todas essas tecnologias usam Node por baixo dos panos. Então, tudo que a gente vai ver aqui hoje, principalmente nesse primeiro módulo, são muitos fundamentos do Node não necessariamente aplicados a um único contexto. A gente não vai começar desenvolvendo um projeto já utilizando um framework, porque eu quero que você tenha a experiência de desenvolver um projeto, entender como o Node funciona por baixo dos panos, como cada coisinha funciona, e a gente vai entrar em coisas inclusive mais profundas do próprio Node. A gente vai entender sobre um dos funcionamentos que eu acho mais incríveis do Node, que é o streaming. Streaming é um processo que a gente tem, uma habilidade que a gente tem quando a gente usa uma tecnologia como o Node, que possui por baixo dos panos uma arquitetura que a gente chama de non-blocking IO, ou seja, traduzindo para o português, input e output não bloqueante, que significa que no Node a gente consegue realizar ações de forma parcial, sem bloquear outras ações de acontecerem ao mesmo tempo. de certa forma, uma concorrência para a gente ter entre os nossos processos. E o que isso permite para a gente, que a gente vai ver já nesse primeiro módulo utilizando o recurso de streaming, é ler, por exemplo, ou escrever dados de forma parcial enquanto a gente continua executando outras tarefas na nossa aplicação.
