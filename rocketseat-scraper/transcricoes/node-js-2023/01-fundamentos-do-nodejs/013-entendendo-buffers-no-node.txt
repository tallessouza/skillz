Sendo bem sucinto, o buffer é uma representação de um espaço na memória do computador, usado especificamente para transitar dados de uma maneira muito rápida, ou seja, os dados armazenados no buffer Eles são armazenados ali para logo serem tratados, ou seja, enviados para algum outro lugar e depois, logo, removidos. Então, são maneiras de a gente conseguir salvar e ler da memória de uma maneira muito performática. E por que muito performática? O Node utiliza esse modelo de buffer na leitura e na escrita de streams porque é mais performático você ler parcialmente uma informação de forma binária, que é como o buffer guarda na memória, ele guarda os dados de uma maneira binária, do que necessariamente um texto, uma string, algo, um hexadecimal, algo que... hexadecimal não, hexadecimal é binário, mas algo que tem muito mais informações, sei lá, acentos, teals, que a gente precisa guardar com encoding, muito mais pesado para ser lido. O buffer existe dentro do Node, foi uma API criada dentro do Node, especificamente pela incapacidade do JavaScript de trabalhar com dados binários de maneira eficiente. O JavaScript, por muito tempo, nunca teve uma forma nativa de trabalhar com dados binários. Hoje em dia até existe uma forma nativa de trabalhar com dados binários, que é essa API aqui chamada Typed Array do JavaScript porém ela ainda não é utilizada pela grande maioria tanto do Node e tal a gente consegue usá-la no browser mas no Node especificamente o Node não usa essa API internamente aqui para trabalhar com dados binários então a gente não consegue usar esse Typed Array a gente precisa usar o Buffer ainda que é a forma nativa que o Node criou para a gente conseguir ler e escrever dados na memória do computador.
