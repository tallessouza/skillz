WEBVTT Uma coisa que a gente vai fazer aqui no nosso projeto, antes de a gente prosseguir aqui adicionando as funcionalidades e tudo mais, é algo que eu gosto de fazer em todo projeto, independente de back-end, front-end, todo projeto que eu trabalho com o JavaScript especificamente, que é configurar, aqui criando um arquivo chamado .npmrc, configurar o próprio npm, que é o gerenciador de pacotes que a gente está utilizando. Porém, esse arquivo aqui, ele é lido também pelo pnpm, yarn ou qualquer outro gerenciador de pacote. Aqui dentro a gente vai passar uma única opção chamada save exact como sendo true. O que essa configuração aqui vai fazer? Quando a gente executar um npm install daqui pra frente, um npm i ou qualquer coisa assim, seja uma dependência de desenvolvimento ou uma dependência de produção, a versão da dependência, que vai ficar aqui no package.json, ela vai ficar fixa. Inclusive o meu, eu coloquei essa configuração global pra não esquecer, mas aqui a gente criou ela a nível de projeto, então você pode, depois de criar esse arquivo, vir aqui e fazer a instalação de todas as dependências que a gente fez antes, de novo. Então, instalar ali o Typescript, Typesnode, TSX, TSAP como dependência de desenvolvimento, e depois o Fastify como produção, e ele vai fixar a versão. Então, você vai ver que nenhuma versão aqui das dependências vai ficar com acento circunflexo, com util ou qualquer coisa assim, indicando que aquela, por exemplo, eu preciso instalar uma dependência no mínimo nessa versão ou qualquer coisa assim, seguindo ali o semantic versioning, que é o que é geralmente utilizado para essas versões de pacotes. Mas por que a gente fez isso? Mais para frente, quando a gente colocar o nosso projeto no ar, uma das coisas que é primordial que a gente faça é que a gente esteja constantemente atualizando, se possível, as dependências do nosso projeto. Então, não é legal, por exemplo, se uma dependência está atualizando, algum motivo tem. Geralmente, por algum problema de segurança, para ficar mais performática, para trazer mais funcionalidades... Então, é normal que, com o passar do tempo no nosso projeto, a gente queira evoluir a versão das dependências, para não ficar numa versão super desatualizada em algum momento a gente querer utilizar alguma funcionalidade e a nossa versão ainda não tem aquela funcionalidade. E aí, não dá para a gente tirar um belo dia da nossa vida e ir lá e atualizar todas as dependências. E a gente vai matar o nosso projeto, certeza, porque várias dependências trazem breaking changes, elas trazem mudanças de API que o nosso projeto, às vezes, vai precisar se adaptar. Então, quando a gente tem a versão exata aqui dentro da dependência, a gente pode utilizar alguns bots do GitHub, como, por exemplo, o próprio Renovate, e isso não é do GitHub especificamente, dá para a gente utilizar nos outros sistemas como GitLab, BitBucket, por aí vai... Esse bot basicamente fica tentando atualizar as dependências do nosso projeto, ou seja, ele vai lá, baixa o nosso projeto e sobe a versão aqui do TSA, por exemplo, sei lá, versão 6.6.4. E ele vai lá e roda os testes da nossa aplicação, sim, os testes automatizados, testes end-to-end, testes unitários, para ver se a nossa aplicação continuou funcionando sem problemas nenhum após essa atualização. Caso os testes passarem, ou seja, a nossa aplicação não teve erros, ele então cria uma pull request aqui dentro do GitHub, uma sugestão de alteração do código, falando que a gente pode atualizar essa versão dessa dependência. Isso é muito legal, a gente conseguir automatizar para que a gente tenha constantemente as versões das nossas dependências atualizadas. E caso os testes falharem, ele vai falar exatamente qual teste falhou e a gente consegue acessar a documentação dessa dependência que a gente está querendo atualizar, para conseguir fazer os ajustes necessários. Então, isso aqui é bem legal. E é apenas uma dica que eu dou para você, assim que você estiver criando seu projeto em JavaScript, sempre salvar as dependências com a sua versão exata dentro do package.json.
