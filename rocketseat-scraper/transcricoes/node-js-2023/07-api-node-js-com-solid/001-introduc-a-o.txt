WEBVTT Fala Dev, bora pra mais um projeto de Node e nesse projeto o nosso foco total é a gente trabalhar com Design Patterns, testes automatizados e ver um pouquinho sobre princípios da programação como o próprio Solid. E a ideia desse projeto vai ser a gente testar esse projeto do começo ao fim, ou seja, uma das coisas que você vai perceber é que desde o começo No primeiro momento que a gente começar a criar alguma funcionalidade, de uma maneira mais profunda, trabalhar as regras de negócio dessa funcionalidade, a gente já vai começar a escrever testes automatizados, começando pelos testes unitários e terminando nos testes end-to-end. Vai ser bem legal, porque a gente vai conhecer um monte de design patterns interessantes que a gente utiliza no dia a dia, como por exemplo, repository pattern, in-memory databases, factory pattern... E a gente vai arquitetar a nossa aplicação de uma maneira muito mais interessante do que a gente vem arquitetando as aplicações anteriores, porque a gente vai começar a trabalhar com inversão de dependência, princípios da programação que deixam o nosso código mais manutenível e também permitem a gente criar testes automatizados de uma maneira muito mais simples. A gente vai se ver bastante durante essas próximas mais de 60 aulas daqui pra frente, do começo do projeto do NPM Init até o envio desse projeto lá pra dentro do GitHub, onde a gente vai também fazer todo o fluxo de integração contínua e deploy contínuo. Te espero lá e a gente se vê ao longo das próximas aulas. Tchau!
