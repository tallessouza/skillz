Como eu havia mencionado, uma das formas mais atuais de você fazer o gerenciamento de dependências do seu projeto hoje no Xcode é utilizando aqui o SPM, que é o Swift Package Manager. Então, esse recurso permite que a gente faça a inserção e a instalação de dependências direto hoje da própria IDE, no caso do Xcode. Mas, se alguém quiser utilizar ainda o modelo do CocoaPods, do gerenciamento de dependências via CocoaPods, basta que você vá até a pasta do seu projeto, né? Dá um pod init aqui, ele vai criar, então, um arquivo pod file pra você. uma vez que ele criou esse arquivo podfile você pode então editar esse arquivo então vou pegar aqui vou editar ele vou colocar aqui as configurações para a gente fazer a inserção das dependências que nós precisamos aí dentro do nosso projeto no caso o kinfish e o snapkit vou dar aqui um salvar E vou lá, então, de novo no meu prompt e dou um pod install. Então, ele vai analisar as dependências, vai fazer o download e a instalação dessas dependências e vai, portanto, criar para mim o arquivo aqui, xcworkspace, que é o arquivo que, agora sim, possui aqui as dependências que a gente precisa para rodar o nosso software utilizando o CocoaPods. Então, fica aí a cargo de cada um utilizar a forma como achar mais conveniente para fazer essa gestão de dependências, beleza? Valeu aí, galera!
