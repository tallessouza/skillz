# Metadados de Paginas no Next.js

> Configure metadata no layout raiz com template e default, e use export const metadata nas paginas para titulos especificos.

## Rules

1. **Defina metadata no root layout** — use `export const metadata` em `layout.tsx` para titulo e descricao globais, porque e o ponto central que todas as paginas herdam
2. **Use template com %s para sufixo/prefixo** — `title: { template: "%s | Product Roadmap", default: "Product Roadmap" }`, porque permite titulo unico por pagina mantendo identidade do app
3. **Sempre defina default** — o campo `default` no title e usado quando a pagina nao exporta metadata proprio, porque evita aba sem titulo
4. **Exporte metadata estatico nas paginas** — use `export const metadata: Metadata = { title: "Board" }` em cada `page.tsx`, porque o Next.js faz merge automatico com o template do layout
5. **Use generateMetadata para dados dinamicos** — quando o titulo depende de fetch (ex: nome do produto), exporte `generateMetadata` async em vez de const, porque permite requisicoes HTTP antes do render
6. **Nunca misture const metadata e generateMetadata** — use um ou outro por arquivo, porque o Next.js ignora um deles silenciosamente

## How to write

### Root layout com template

```typescript
// app/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: {
    template: "%s | Product Roadmap",
    default: "Product Roadmap",
  },
  description: "Follow the development progress of our entire platform.",
};
```

### Pagina com metadata estatico

```typescript
// app/board/page.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "Board", // Resultado na aba: "Board | Product Roadmap"
};
```

### Pagina com metadata dinamico

```typescript
// app/products/[id]/page.tsx
import type { Metadata } from "next";

export async function generateMetadata({
  params,
}: {
  params: { id: string };
}): Promise<Metadata> {
  const product = await fetch(`/api/products/${params.id}`).then((r) =>
    r.json()
  );

  return {
    title: product.name, // Resultado: "Product Name | Product Roadmap"
  };
}
```

## Example

**Before (default do create-next-app):**
```typescript
// app/layout.tsx
export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};
```

**After (com template e identidade):**
```typescript
// app/layout.tsx
export const metadata: Metadata = {
  title: {
    template: "%s | Product Roadmap",
    default: "Product Roadmap",
  },
  description: "Follow the development progress of our entire platform.",
};

// app/board/page.tsx
export const metadata: Metadata = {
  title: "Board",
};
// Aba mostra: "Board | Product Roadmap"
```

## Heuristics

| Situacao | Faca |
|----------|------|
| Todas as paginas compartilham sufixo no titulo | Use `title.template` com `%s` no root layout |
| Pagina tem titulo fixo conhecido | Use `export const metadata` |
| Titulo depende de dados do servidor (produto, user) | Use `generateMetadata` async |
| Pagina nao precisa de titulo proprio | Nao exporte metadata, o `default` do layout sera usado |
| Precisa de OpenGraph, Twitter, Robots | Adicione no mesmo objeto metadata (layout ou page) |

## Anti-patterns

| Nunca faca | Faca em vez disso |
|------------|-------------------|
| `<title>` manual no JSX | `export const metadata` ou `generateMetadata` |
| Titulo hardcoded sem template | `title: { template: "%s \| App", default: "App" }` |
| `generateMetadata` para titulo fixo | `export const metadata = { title: "Board" }` |
| Ambos `const metadata` e `generateMetadata` no mesmo arquivo | Escolha um: estatico OU dinamico |
| Deixar o default "Create Next App" | Substitua pelo nome real do projeto |

## Troubleshooting

### Meta tags nao aparecem no preview de compartilhamento
**Symptom:** Ao compartilhar link no WhatsApp/Twitter/LinkedIn, preview aparece sem imagem ou descricao
**Cause:** Falta de tags Open Graph ou tags com valores vazios/incorretos
**Fix:** Adicionar `og:title`, `og:description`, `og:image` via metadata export ou generateMetadata. Verificar com https://cards-dev.twitter.com/validator

### Title duplicado ou generico no Google
**Symptom:** Google mostra title diferente do configurado ou igual para todas as paginas
**Cause:** Title identico em todas as paginas ou faltando configuracao especifica por rota
**Fix:** Configurar metadata unica por pagina usando `export const metadata` ou `generateMetadata` com dados dinamicos
