# O Que É Ser Dev e O Que É Programar

> Programar é resolver problemas do mundo real através de linhas de código — a mentalidade de resolução de problemas vem antes da sintaxe.

## Key concept

Ser desenvolvedor significa ter a capacidade de se comunicar com o computador usando linguagens que ele entende, dando ordens para que ele execute tarefas e resolva problemas reais. A habilidade central não é memorizar sintaxe — é desenvolver uma mentalidade orientada a problemas. Cada aplicativo que existe (Instagram, iFood, Uber) nasceu de alguém identificando um problema real e resolvendo com código.

## Decision framework

| Quando você encontrar | Aplique |
|----------------------|---------|
| Um problema novo para resolver | Decomponha em passos menores antes de codar |
| Frustração com bugs ou erros | Lembre: problemas são o trabalho, não exceções |
| Dúvida se vale aprender a programar | O mundo depende cada vez mais de software — oportunidades só crescem |
| Tentação de decorar código sem entender | Foque no problema que o código resolve, não na sintaxe isolada |

## How to think about it

### Programar = dar ordens ao computador

O computador é uma ferramenta poderosa mas burra — ele faz exatamente o que você manda. Aprender a programar é aprender a dar instruções precisas, passo a passo, para que o computador execute tarefas por você. A linguagem de programação é o idioma comum entre você e a máquina.

### Problemas são o produto, não o obstáculo

Durante os estudos você vai encontrar problemas. No mercado de trabalho as pessoas vão trazer problemas. Isso não é um bug do processo — é literalmente a descrição do trabalho. Ajuste sua mentalidade: resolver problemas é o que você faz, não algo que atrapalha o que você faz.

### Todo aplicativo nasceu de um problema real

Instagram: pessoas queriam compartilhar fotos. iFood: pessoas queriam pedir comida sem sair de casa. Uber: pessoas precisavam de transporte acessível. Quando olhar para qualquer software, pergunte: "qual problema do mundo real isso resolve?"

## Common misconceptions

| Pessoas pensam | Realidade |
|---------------|-----------|
| Programar é escrever código o dia todo | Programar é pensar em soluções — o código é apenas a ferramenta de execução |
| Preciso ser gênio em matemática | Precisa ter mentalidade de resolver problemas, passo a passo |
| Programação vai ser substituída | O mundo depende cada vez mais de software — a demanda só cresce |
| Saber a sintaxe de cor é o mais importante | Entender o problema e saber decompor em passos é mais valioso |

## When to apply

- Ao iniciar qualquer projeto novo: comece pelo problema, não pela tecnologia
- Ao se sentir perdido em um exercício: volte ao problema original que precisa resolver
- Ao escolher o que estudar: priorize o que resolve problemas reais
- Ao explicar programação para leigos: "resolvo problemas do mundo real com código"

## Limitations

- Este modelo mental é fundacional — não substitui conhecimento técnico específico
- Saber que "programar é resolver problemas" não ensina COMO resolver cada problema
- A mentalidade precisa ser combinada com prática deliberada de código
