# Deep Explanation: O Que É Ser Dev e O Que É Programar

## O raciocínio do instrutor (Mayk Brito)

### A definição central

Mayk define programar de forma direta: **"resolver problemas do mundo real através das linhas de código"**. Ele enfatiza que essa é a essência — não a linguagem, não o framework, não a ferramenta. O problema vem primeiro.

### A mentalidade como pré-requisito

O ponto mais forte da aula é que Mayk não começa falando de tecnologia. Ele começa falando de **mentalidade**:

> "Significa que você tem que ter a mentalidade ajustada para sempre resolver problemas."

Isso é intencional. Muitos iniciantes entram na programação focados em "qual linguagem aprender" quando deveriam focar em "como pensar em soluções". A linguagem é o meio, não o fim.

### A analogia da comunicação

Mayk usa uma analogia poderosa: programar é **aprender a falar com o computador**. Assim como você aprende inglês para se comunicar com falantes de inglês, você aprende uma linguagem de programação para se comunicar com o computador.

A diferença: o computador é literal. Ele faz exatamente o que você manda. Se você der uma instrução errada, ele executa errado — sem julgamento, sem "será que ele quis dizer outra coisa?". Por isso a precisão importa.

### O passo a passo como conceito fundamental

> "O passo a passo é fazer o computador trabalhar para você, fazer o computador executar aquelas tarefas chatas."

Aqui Mayk introduz implicitamente o conceito de **algoritmo** sem usar a palavra. Um programa é uma sequência de passos que o computador segue. Essa ideia de "passo a passo" é o fundamento de toda programação.

### Exemplos do mundo real como motivação

Mayk lista exemplos concretos para ancorar o conceito abstrato:

- **A própria aula** — a plataforma de ensino é software
- **Instagram** — resolver o problema de compartilhar fotos
- **iFood** — resolver o problema de pedir comida
- **Uber** — resolver o problema de transporte

Cada exemplo segue o padrão: **problema real → solução via código**. Isso reforça que programação não existe no vácuo — existe para servir pessoas.

### A visão de mercado

Mayk fecha com uma perspectiva de carreira:

> "Daqui pra frente não tem como viver no mundo de hoje sem programar, sem ter programa, sem ter pessoas que falam com o computador e resolvam os problemas da gente."

Não é hype — é observação factual. Cada setor da economia depende cada vez mais de software. Saúde, educação, transporte, alimentação, finanças — todos precisam de desenvolvedores.

### O que torna esta aula valiosa

A aula é curta mas estabelece o **frame mental** correto para todo o resto do curso. Um aluno que internaliza "eu resolvo problemas com código" vai:

1. Não se frustrar com bugs (problemas são o trabalho)
2. Não se perder em syntax wars (a linguagem é meio, não fim)
3. Não duvidar do valor da profissão (o mundo precisa disso)
4. Abordar cada exercício como um problema a ser decomposto

Essa base mental é mais valiosa que qualquer tutorial de sintaxe.
