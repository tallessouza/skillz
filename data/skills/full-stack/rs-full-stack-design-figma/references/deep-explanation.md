# Deep Explanation: Design - Figma

## Por que o Figma no fluxo de desenvolvimento?

O Figma é uma ferramenta **colaborativa** de design de interfaces. No contexto de um curso full-stack, ele serve como a **fonte de verdade visual** — o desenvolvedor consulta o Figma para saber exatamente como a interface deve ficar.

## Por que duplicar projetos?

O instrutor enfatiza o fluxo de **duplicação** por uma razão prática: o projeto original é compartilhado em modo leitura. Ao duplicar:

1. **Você tem controle total** — pode mover, editar, excluir elementos
2. **Não afeta o original** — outros alunos continuam com acesso ao projeto intacto
3. **Fica na sua pasta** — fácil de encontrar depois

Esse padrão é o mesmo usado em equipes reais: o designer compartilha o arquivo, o dev duplica para fazer anotações, medir espaçamentos, extrair cores e assets.

## Figma como ferramenta colaborativa

O termo "colaborativo" usado pelo instrutor significa que múltiplas pessoas podem trabalhar no mesmo arquivo simultaneamente. No contexto do curso:

- O instrutor cria o design
- Compartilha o link com os alunos
- Cada aluno duplica para sua pasta
- Cada aluno pode fazer modificações independentes

## Progressividade

O instrutor menciona que "conforme as aulas vão passando, você vai entendendo mais detalhes sobre ela". Isso indica que o Figma será usado ao longo do curso como referência para:

- Extrair medidas (padding, margin, font-size)
- Identificar cores e tipografia
- Entender a estrutura de componentes
- Verificar estados de interação (hover, active, disabled)

## Fluxo mental do desenvolvedor com Figma

```
1. Receber link do design
2. Duplicar para pasta pessoal
3. Abrir design duplicado
4. Inspecionar elemento por elemento
5. Implementar no código
6. Comparar resultado com o design
7. Ajustar até ficar fiel
```
